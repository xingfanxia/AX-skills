# 设计原则:这套方法论不止用于英语(泛化边界)

> AX 确认(2026-06):核心逻辑"低动机 + 理解/胜任优先 + AI 降维 + 反背诵 + 嫁接已有娱乐 + 陪伴非监工 + 极小起步 + 零评判进度"不局限于英语,可推广到任何"学习者觉得自己学不好/没兴趣/怕痛苦"的技能习得。
> 这条原则约束所有产品设计:**coaching 方法论按领域无关(domain-agnostic)设计,英语只是第一个 vertical playbook(旗舰垂类),不要把方法论硬编码进英语。**

## 什么能泛化(方法论层 — 领域无关)

这些原则对编程、数学、健身、乐器、绘画、考证、学一门新工具…… 几乎任何"被判过死刑"的学习领域都成立:

| 原则 | 为什么领域无关 |
|---|---|
| 降焦虑优先 / affective filter | 习得性无助是对"我学不好 X"的归因,X 是什么不重要 |
| 胜任优先,产出后置 | 先"看懂/认得出/能跟着做"再"独立产出"——识别先于生成,适用于任何技能 |
| **AI 降维成 i-1 可理解材料** | AI 能把任意内容(代码/数学证明/乐理/菜谱/论文)改写到学习者当前水平——这是最通用的杠杆 |
| 反死记硬背,靠高频"撞见"自然习得 | 间隔/情境化暴露胜过填鸭,对大多数技能成立 |
| 嫁接到已有行为(habit stacking) | 把学习寄生到他已经在做的事上,与领域无关(Fogg B=MAP) |
| 陪伴非监工(SDT relatedness) | 自主/胜任/归属是通用心理需求 |
| 极小起步 + 零评判进度("今天懂了"清单) | 通用习惯科学(Lally / Tiny Habits) |
| 工具型动机:把"学技能"嫁接到他真想要的结果上 | 通用——动机来自"我想要的东西",不是技能本身 |

## 什么不能泛化(英语/语言特有层)

这些是英语这个垂类的"playbook"内容,换领域要换:

- 浏览器**翻译/降维插件**——本质是语言工具(translator),只服务语言/阅读类垂类。
- 听力降级、phonics、连读弱读——语言特有。
- 词汇覆盖率阈值(Hu&Nation 95–98%)——语言特有(松散地对应"低于某个已知基元密度就无法靠上下文推断",可弱泛化)。
- "2026 中文 AI 生态成熟 → 英语信息边际收益只在 4 窄场景"——英语+信息获取特有的动机校正。

## 由此推出的产品架构(三层)

```
┌─ 方法论层(领域无关) ── coaching skill 的内核:诊断 + 8 原则 + 陪伴话术 + 进度仪式
│      ↓ 实例化
├─ 垂类 playbook 层 ──── 英语(旗舰)/ 编程 / 健身 / 乐器 / 考证 … 每个一份 playbook
│      ↓ 调用
└─ 工具层 ───────────── 语言特有:翻译/降维浏览器插件;其他垂类有各自工具
```

含义:
- **coaching skill 应是 domain-agnostic 的内核 + 可插拔 vertical playbook**;英语是 `playbooks/english`,不是写死的主体。
- **浏览器插件是英语(及阅读类垂类)的工具层**,不要求泛化——它就是个为学习者重做的 translator。
