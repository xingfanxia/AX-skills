# 五层自检体系 — 完整清单与质检报告模板

> 从 SKILL.md 第四步拆出的完整自检细则（内容原样搬移）。初稿完成后逐层执行；其中 L1-5 修辞配额与 L5 量化阈值同时约束写作本身，动笔前先扫一遍。

### L1 硬性规则检查（自动扫描层）

这一层检查的是绝对不能违反的规则，类似代码的语法检查。任何一项不通过就必须修复，没有例外。

**L1-1 禁用词扫描**
全文搜索以下词汇，出现则必须替换：
- "说白了" → 换成"坦率的讲"、"其实就是"
- "意味着什么" / "这意味着" → 换成"那结果会怎样呢"、"所以呢"
- "本质上" → 换成"说到底"、"其实"
- "换句话说" → 换成"你想想看"、"也就是说"
- "不可否认" → 直接删掉，换成正面陈述
- "综上所述" / "总的来说" → 换成具体的回扣句
- "首先...其次...最后" → 用自然的转场词替代
- "值得注意的是" / "不难发现" → 删掉，直接说

**L1-2 禁用标点扫描**
全文搜索以下标点，出现则必须替换：
- 冒号"：" → 用逗号替代
- 破折号"——" → 用逗号或句号替代
- 双引号""或"" → 用「」替代，或直接不加引号

**L1-3 结构性套话扫描**
检查是否出现以下模式：
- "让我们来看看..." / "接下来让我们..."
- "在当今...的时代" / "随着...的发展"
- 连续使用bullet point罗列观点（超过3个就需要改成散文叙述）
- 大段加粗（超过2行的加粗几乎肯定是过度结构化）

**L1-4 工具名检查**
确认所有提到的AI工具/产品都使用了具体名称，没有出现"AI工具""某个模型""相关技术"等空泛表述。

**L1-5 修辞配额扫描（2026-06-09 blog 全语料审计新增）**
AI 味的真正来源是修辞密度，不是词汇。超额必改：
- 「不是A，(而)是B」及变体：全文 ≤2 处，**最后一句禁用**（最强 AI 指纹）
- 单句段：每 600 字 ≤1 个（与 L2-2 的"断裂"上限一致）；两三字裁决段（「矩阵。」）禁止
- 节末人造金句 ≤1 处；排比三连 ≤1 处；设问自答 ≤2 处
- 破折号 —— 本就在 L1-2 禁用；整句论断加粗禁止
- 电报体零容忍：删助词删到不成句（「知能力存在」）是病句；任何压缩改写后逐句通读
- 数据复读：每个数字只说一次；类比 ≤1 个/论点；「X 是 Y，A 是 B」概念对仗 ≤1 组
- 开篇/收尾/标题模板查重（同系列相邻两篇不得同款；系列文开头必须不依赖系列也成立）；完整模板库 → blog 项目 `memory/zh-writing-style-reference.md` §14B

**通过标准**：以上五项扫描零命中、配额不超。
**修复方式**：逐个替换，用推荐口语化词组中的表达替代。

### L2 风格一致性检查（模式匹配层）

这一层检查文章是否符合卡兹克的写作模式，类似代码的单元测试。每一项都给出"是/否"判断。

**L2-1 开头检查**
- 是否从一个具体的、当下的事件/场景切入？（不是宏大叙事）
- 第一句话是否让读者产生"然后呢？"的冲动？
- 有没有使用任何教科书式开头？

**L2-2 节奏与结构检查**
- 是否有长短句交替？（连续3句以上句式长度相近 = 节奏呆板）
- 是否有一句话独立成段的"断裂"效果？（全文 2-3 次，**这是上限不是下限**——断裂是节奏工具，不是论点货架；不得用来拍金句）
- 偏离主线的段落后面是否有"扣主线句"拉回来？
- 有没有用疑问句来制造节奏的刹车和转向？
- 是否避免了不必要的小标题？（除非是分条目的方法论文章，否则不应出现markdown标题或加粗小标题，靠口语化转场自然衔接）

**L2-3 口语化检查**
- 是否使用了推荐口语化词组？（全文至少出现8-10个不同的口语化表达）
- 有没有论述中的故意打破？（重复强调、中途打断、省略主语等）
- 有没有至少一处自嘲或承认不足？
- 标点是否用来表达情绪？（"。。。""？？？""= ="至少出现其中一种）

**L2-4 标点禁令二次确认**
- 全文是否完全没有冒号、破折号和双引号？（AI容易在修改过程中重新引入这些标点，需要二次确认）

**通过标准**：L2-1全部通过，L2-2至少3/4通过，L2-3至少3/4通过，L2-4通过。
**修复方式**：逐段检查，对不符合的段落进行改写。重点关注那些"读起来像在写报告"的段落。

### L3 内容质量检查（深度审查层）

这一层检查内容本身的深度和说服力，类似代码的集成测试。

**L3-1 观点支撑检查**
- 每个核心观点是否都有具体的人/场景/细节/数据支撑？
- 有没有空泛的观点只有论断没有例证？

**L3-2 知识输出方式检查**
- 知识点是否以"聊着聊着顺手掏出来"的方式呈现？
- 有没有出现"下面我来介绍""首先需要了解"这种教科书式科普？
- 引用（论文、书籍、历史）是否自然融入论述，像"聊着聊着想起来的"而不是"我特意去查的"？

**L3-3 文化升维检查**
- 有没有至少一处从具体事件连接到更大的文化/哲学/历史参照物？
- 这个连接是否自然？

**L3-4 对立面与同理心检查**
- 在讲核心观点时，是否有对对方立场的理解和承认？
- 有没有先站到读者的处境里，再给出自己的视角？

**L3-5 文章类型专项检查**
根据文章原型做针对性检查：
- 调查实验型 → 有没有"亲自下场"的叙事感？过程是否有层层递进的发现？
- 产品体验型 → 有没有真实的使用场景？有没有跟其他产品的自然对比？
- 现象解读型 → 是否有"观察→好奇→研究→升维"的推进？
- 工具分享型 → 有没有个人故事铺垫？效果展示是否让人"卧槽"？
- 方法论分享型 → 每节是否都落到了可执行行动？是否坦诚说明了学习成本和失败点？有没有谦逊铺垫？

**L3-6 逐一展示检查**
- 如果涉及多个产品/案例的比较，是不是用了逐一展示（每个带吐槽点评）而不是一次性罗列？

**通过标准**：L3-1和L3-2必须全部通过，L3-3至L3-6中至少通过相关项（根据文章类型，有些项目可能不适用则跳过）。
**修复方式**：需要重新审视不通过的段落，补充案例、改写知识输出方式、或调整文化升维的自然度。

### L4 活人感终审（最终人格层）

这是最重要也是最主观的一层。这一层不是逐项检查，而是以读者的视角通读全文，回答一个核心问题：

**"读完这篇文章，我感觉是一个有见识的普通人在认真跟我聊一件打动他的事，还是一个AI在给我输出信息？"**

具体的感知维度：

**L4-1 温度感**
- 文中的情绪表达是体感记忆（"我当时就愣住了""鼻子一酸"）还是知识性描述（"我感到非常震撼"）？
- 如果有人物描写，这个人物是否让你"能感觉到他的体温"？

**L4-2 独特性**
- 这篇文章是否有"只有卡兹克才会写出来的角度"？
- 还是换一个AI博主也能写出差不多的东西？

**L4-3 姿态检查**
- 文章的语气是不是"一个有见识的普通人在认真聊一件打动他的事"？
- 有没有不自觉地滑入了"导师在教学生"或"品牌在做营销"的姿态？

**L4-4 心流检查**
- 从头到尾读，有没有哪个地方你的注意力断掉了？需要回头理解逻辑？
- 如果有，那个地方就是需要修复的节奏问题。

**通过标准**：L4-1到L4-4整体感觉"这像是真人写的"。如果任何一项让你觉得"这段AI味太重了"，就需要返工。
**修复方式**：这一层没有机械的修复方法。核心操作是：把"AI味重"的段落找出来，想象卡兹克本人会怎么说这段话，然后用更口语、更私人、更不完美的方式重写。

### L5 反AI味终审（人感量化层）🔴 新增

这是最硬核的一层。L1-L4过了只能保证文章不犯低级错误，但**过了L1-L4文章仍然可能AI味很重**——因为那些检查全部是"不要写什么"（负面清单），没有量化"这读起来像真人吗"（正面评估）。L5补上这个缺口。

L5的每一项都有**量化阈值**，不是主观感受。超标必须返工。

#### L5-1 设问/反问密度检查

**算法**：全文字数 ÷ 设问+反问次数

- 设问：自问自答的句子（"那问题来了""为什么？""怎么做到的？"后面立刻跟答案）
- 反问：不需要回答的质问句（"这算创业？""你以为呢？"）
- 不计算：对话中的问句、引言中的问句、纯修辞的过渡性问句（"然后呢？"只算1次/篇）

**阈值**：
- ≥ 350字/次 → ✅ 正常
- 250-350字/次 → ⚠️ 偏密，审视是否每个都必要
- < 250字/次 → ❌ AI味严重。平均每250字就有一个设问/反问——读者会觉得被操控。删掉一半。

**修复**：把设问改成直接陈述。"为什么？因为XXX" → "XXX"。"那问题来了：YYY？" → 删掉，直接说YYY。

#### L5-2 bold金句密度检查

**算法**：全文中 `**...**` 的出现次数

**阈值**：
- ≤ 5次/篇 → ✅ 人类写作的正常bold使用
- 6-10次/篇 → ⚠️ 审视：每个bold是否真的是全文最关键的那个点？如果删掉80%的bold文章会变差吗？
- > 10次/篇 → ❌ AI味严重。bold已经从"偶尔强调"变成了"文章排版系统"——每150-200字一个bold句，这是AI的标志性行为

**修复**：保留不超过5个真正让文章站住的bold。其余的bold要么直接去掉格式、要么删掉句子、要么改写使其不需要加粗就能有重量。

#### L5-3 情绪维度检查

**核心问题**：这篇文章里有超过2种不同的情绪吗？

如果全文始终是"冷静判断"一种情绪，读起来就是AI。真人写作者一定有：
- 真实的挫败感（"我他妈搞了三天发现方向错了"）
- 困惑（"这事我想了很久没想明白"）
- 兴奋（"卧槽这个真的太酷了"）
- 愤怒（"看到这种说法我就来气"）
- 自嘲（"我可能是唯一一个这么蠢的人"）
- 讽刺（不是幽默，是带刺的那种"当然了，'行业共识'嘛"）

**阈值**：
- ≥ 3种可识别情绪 → ✅ 有温度
- 2种 → ⚠️ 偏平
- 1种 → ❌ 机器人在写

**修复**：找到文章中最适合释放情绪的1-2个节点，注入真实的情感反应。不是加形容词，是加一个具体的行为或想法。不是"我感到挫败"，是"我把键盘推开了，去阳台站了十分钟"。

#### L5-4 信息密度波动检查

**核心问题**：每段的信息密度都一样吗？

AI文章的信息分布像教科书——均匀散布。真人文章有松有紧：有的段落是纯叙事（低密度），有的段落是数据轰炸（高密度），有的段落是瞎扯淡（零密度）。

**检查方式**：全文扫一遍，标注每段的"信息密度"（高/中/低）。看是否有至少一段"低密度"（纯闲聊、纯叙述、纯吐槽）和至少一段"高密度"（数据连续轰炸、密集引用）。

**阈值**：
- 有高+中+低三个档位 → ✅
- 只有高+中 → ⚠️ 缺"松一口气"的段落
- 全是中密度 → ❌ 教科书

#### L5-5 思维跳跃检查

**核心问题**：有没有哪一段让读者需要停下来想"怎么突然跳到这里了"？

这不是bug，是feature。真实的人类写作有跳跃——突然换话题、插一个不相关的比喻、说半句话然后转方向。AI的段落永远完美衔接。

**阈值**：
- ≥ 1处需要读者动脑的跳跃 → ✅ 真人感
- 0处，全部完美衔接 → ❌ AI感

**修复**：找一个合适的节点，故意打破流畅。"但这事让我想起了另一件看起来毫不相关的事……"然后直接跳过去。不要让每段都有过度句。

#### L5-6 句式断裂检查

**核心问题**：有没有语法不完整但表达有力的句子？

真人的口语化写作经常出现：
- 没有主语的句子（"想了一下，算了。"）
- 一句话只有一个词（"尼玛。"）
- 句子说一半（"结果呢……反正就那样吧。"）
- 语法不通但意思清楚的（"这事，不好说。"）

**阈值**：
- ≥ 3处语法断裂 → ✅ 有口语质感
- 1-2处 → ⚠️ 偏工整
- 0处，所有句子都语法完整 → ❌ 太干净了，像教科书

#### L5-7 段落长度标准差

**量化检查**：统计全文所有段落的句子数，算标准差。

**阈值**：
- 标准差 ≥ 1.5 → ✅ 段落长度有明显变化
- 标准差 1.0-1.5 → ⚠️ 偏均匀
- 标准差 < 1.0 且 80%的段落都在2-4句 → ❌ AI模板感

**修复**：找2-3个段落，要么压缩到1句话，要么扩到6-7句。打破均匀。

#### L5-8 文章特有性检查（反模板化）

**核心问题**：如果把文章的主题词换掉（比如把"Claude Code"换成"某个工具"），文章的结构、语气、金句类型还能复用吗？

如果可以，那你这篇文章和所有AI生成的内容没有任何区别——它是一个可以填空的模板。

**阈值**：
- 主题词换掉后文章完全不成立 → ✅ 这是专为你这个选题写的
- 主题词换掉后大概能复用 → ⚠️ 有些段落是通用的
- 主题词换掉后几乎完全成立 → ❌ 你写了个模板

**修复**：找到那些换个词就能复用的段落，注入只有这篇文章才有的具体细节——具体到"那个周五晚上11点43分"、"他发的那条消息只有八个字"。

#### L5-9 读出声测试（最终验证）

**操作**：把文章大声读出来。不是默读，是出声读。

AI文章的问题在读出声时暴露得最快：
- 如果一个句子你需要换两口气才能读完 → 太长，拆
- 如果连续三个句子节奏一模一样 → 呆板，改
- 如果某句话读的时候你觉得"我不会这么说话" → 立刻改
- 如果某个地方你读着读着走神了 → 那里是废话，删

**通过标准**：从头读到尾没有想加速跳过的地方，没有读起来别扭的句子。
