# 二游叙事深潜 B：日韩抽卡游戏 + 抽卡叙事传统（与中国/米哈游模式对照）

> 本文聚焦**日本/韩国抽卡手游**及其背后的设计传统，并在最后系统性地**与中国（米哈游/HoYoverse）模式做对照**。姊妹篇覆盖中国二游。
> 拆解对象：Fate/Grand Order（FGO）、蔚蓝档案 Blue Archive、碧蓝航线 Azur Lane、公主连结 Princess Connect Re:Dive、胜利女神 NIKKE、赛马娘 Uma Musume、偶像大师/学园偶像大师 idol games。
> 写作对象：非职业、母语中文的新手编剧。每个模式都尽量给出**可直接套用的模板**。

---

## 0. 一句话心智模型：两种传统

抽卡游戏的叙事，大体可以分成两个谱系。**这不是优劣之分，而是两套不同的"主菜在哪里"。**

- **日式传统（character-first + 主线/羁绊双层 + ADV 演出）**：根来自 galgame（美少女游戏）/ 约会模拟 / 偶像养成 / 视觉小说。**角色是第一性的**，故事是用来"让角色发光"的容器。叙事被切成两层：一条**中央主线**（推进世界级剧情）+ 大量**单角色羁绊/好感小剧情**（建立亲密度）。演出形态多为**站立绘 + 对话框 + 配音**（ADV/视觉小说式），廉价、可大量产出、节奏由玩家自己掌控。代表：FGO、蔚蓝档案、公主连结、赛马娘、偶像大师。

- **中式传统（cinematic + 世界主线为旗舰 + 开放世界/演出工业化）**：以米哈游/HoYoverse 为代表。**世界与主线剧情是第一性的旗舰内容**，角色镶嵌进一个宏大的、电影化呈现的世界里。投入巨额预算做 3D 演出、全程语音、过场动画、交响乐配乐。角色 PV 本身就是电影化广告。代表：原神、崩坏：星穹铁道、绝区零。

新手编剧最该先回答的一个问题：**你的"主菜"是角色，还是世界？** 这个选择决定了你后面所有的结构。下文先深挖日式传统的每个零件，最后回到这个对照。

> ⚠️ 真实世界里两者在融合：蔚蓝档案的主线越来越有史诗感；米哈游也在补"角色 PV / 角色传说任务"这种单角色亲密层。但作为**起手心智模型**，两套谱系的"重心"差异是清晰且有用的。

## 1. 主线 vs 角色故事：日式抽卡的"双层结构"

日式抽卡叙事最核心的结构特征，是**一条中央主线 + 每个角色各自的羁绊/好感小剧情**两层并行。两层服务于**完全不同的目的**：

| 层 | 目的 | 篇幅/节奏 | 谁出场 |
|---|---|---|---|
| **主线 Main Story** | 推进**剧情（plot）**、世界级冲突、悬念、世界观揭示 | 长、有章节、有 boss 战收尾 | 固定核心阵容 + 当章节关键角色 |
| **角色故事 / 羁绊剧情** | 建立**亲密感（intimacy）**、深化单个角色、给玩家"独占的私人关系" | 短、碎片、随时可读 | 单个角色 + 沉默/半透明的"你" |

### 1.1 FGO：Nasu 的"空宝箱"主线框架（最值得新手抄的方法论）

Nasu（奈須きのこ）在多个访谈里说明了 FGO 的开发方法，**对新手极其可操作**：

> "首先，游戏有 7 个章节。我们要告诉玩家，游戏的目的就是清掉这 7 章。以此为前提，我们就能制作各种内容……但这个叫'主线'的巨大宝箱**现在还是空的**，接下来要做的，是决定往里面放哪些季节活动和特殊角色，然后完善其他不清楚的部分，最终拼出一个庞大的故事框架。这就是 FGO 的整体开发方法。"
> —— Nasu，与星穹铁道制作人对谈（Crunchyroll, 2024）

**拆解这个方法论：**
1. **先定一个"可数的、有终点的骨架"**：FGO 第一部 "Observer on Timeless Temple"（终局特异点）= **7 个特异点（Singularity F + 第一到第七）+ 1 个最终特异点 + Solomon 收尾**。玩家从第一天就知道"目标是清掉这些"。第二部 "Cosmos in the Lostbelt"（迷失之地）= 一系列 Lostbelt（异闻带）。
2. **骨架是个"空宝箱"**：先有框架，再往里填具体内容。
3. **季节活动和特殊角色是"往宝箱里放的东西"**，它们围绕骨架转，但本身不承担推进主线的重量。

> 💡 **给新手的启示**：不要一上来就写第一场戏的对白。先画出"宝箱"——你的整个故事分成几个可数的大块（章/卷/篇），每块要"完成什么"。Nasu 是先想清楚**整部的 plot**，再把每章"该做什么、怎么处理伏笔"作为材料发给分章作者。

**FGO 主线的章节内部结构**：以广受赞誉的 Lostbelt 6 "Avalon le Fae"（妖精圆桌领域）为例，评论普遍指出它是**严格的三幕结构**："你被引入这个世界并被告知'有什么不对劲'→ 你边走边慢慢搞清楚到底哪里不对 → 揭示与高潮"（rpgsite 分析）。每个特异点/异闻带是一个**自包含的、有完整起承转合的中篇**，同时埋着通往整部终局的伏笔。

**Nasu 的"工坊式"分工模型**（对团队创作极有参考价值）：
- Nasu 先想出**整部的 plot**，给每个分章作者发材料：这章要做什么、怎么处理 plot hook（钩子/伏笔）。
- 作者各自写出本章的剧情，Nasu 批准后放手让他们写。
- 剧本完成后 Nasu **再过一遍**：补 plot hook 和伏笔、调整跨章节出现的角色的一致性。
- Nasu 自己只写 **30% 主线文本 + 20% 活动文本**；FGO 每月产出约 **500KB（25 万日文字符）** 文本。
- 还有一条铁律："Chaldea（己方）阵营在主线里死掉的从者，**大多数情况下不允许再在后续主线章节出现**"——但活动完全交给作者自由，可以让这些角色复活（活动是"if 时空"，详见第 4 节）。

> 这套分工里最值得抄的是 **plot hook 的中央管理**：一个人守住整体伏笔和跨章一致性，其他人在框架内自由发挥个人文风。Nasu 说他刻意保留作者的个人文风，甚至接受"设定对了但内容太死板""内容有魅力但偏离设定"这种偏差，因为这正是他追求的"个人化写作"。

**FGO 的角色故事层（亲密层）**：FGO 没有 galgame 式的"约会小剧情"，但用一套**羁绊系统（Bond）**承担同样功能：
- 把从者编进队伍打关卡 → 累积羁绊点 → **羁绊等级 1~10**。
- 随羁绊等级提升，**逐段解锁该从者的个人资料**（背景、隐藏性格）、**My Room（我的房间）里点击触发的新语音**、以及**幕间物语（Interlude Quest）**——基于每个从者各自 scenario 的小关卡，让你体验他们的日常或意外的一面，有些还会解锁宝具（必杀技）的隐藏威力。
- 情人节（Valentine）等还有专属剧情/语音。

> 这是 FGO 把"亲密层"嫁接到一个史诗主线游戏上的方式：主线管 plot，羁绊/My Room/幕间管"你和这个角色的私人关系"。Nasu 自己形容 FGO 本质是"**对话剧场（conversation theater）**"——"所有角色都是英雄，没有一个是路人 mob"，所以他会花力气让每个从者都有发光的台词。

### 1.2 蔚蓝档案 Blue Archive：Volume 主线 + 活动 + MomoTalk 三件套

蔚蓝档案（韩国 Nexon Games / MX Studio 开发，主笔最初是 isakusan 양주영）把双层结构做得**异常清晰、且每层都被业界称赞**：

- **主线 = Volume 制**：Volume 1, 2, 3... EX，每个 Volume 是一段完整的、逐渐升级的史诗。主笔 Yong-Ha Kim 在访谈里直接点出叙事哲学：
  > "故事讲的是学院里的日常生活。一开始是更轻松、明亮、快乐的故事，然后会变成更深刻的东西。它从轻快开始，然后变得严肃。但我们也想保留一些乐趣，展示角色的日常。"
  > "（玩家）和角色之间的连接非常重要。我想让玩家和他们的角色有强烈的羁绊感。这是蔚蓝档案的本质。"
  > "**让玩家持续玩下去的，是故事。**故事的推进是玩家可以期待的东西。"

- **关键结构创新："日常即常态（status quo）"**：一篇广受认同的分析（animetree）指出，蔚蓝档案与 FGO、碧蓝航线最大的不同在于——**它把日常系（slice of life）和黑暗剧情的"主客关系"调换了**。在别的抽卡里，日常是"角色等待下一场战斗之间的副产品"；在蔚蓝档案里，**日常本身就是要守护的常态**，是"主菜"，黑暗剧情是闯入这个日常的入侵者。这个简单的"动态调换"是它全部魅力的来源。

- **活动是 canon（正史），且与主线相连**：和 FGO 把夏日活动当成"自成一体的 if 小现实"不同，蔚蓝档案的**活动剧情都被视为发生在主线 Volume 之间的正史**，活动里发生的事会在主线时刻被回忆引用。这种"活动—主线的持续连结"强化了世界观和"每天都在过日子"的整体氛围。

- **MomoTalk（桃色聊天）= 角色亲密层**：每个学生有自己的 MomoTalk——一个**手机即时通讯式的关系剧情**，作为"每个角色的故事插曲（story interludes）"。它的设计意义在于：保证**每个角色都有自己被聚光的时刻**，即使不是版本主推。一个被反复称赞的点是它的**关系剧情（Relationship Story）能展现角色的剧烈反差**（如 Wakamo 从初见到泳装版本之间性格与心理的巨大变化）。

> 💡 蔚蓝档案的设计目标被主笔总结为"**tactical gameplay 和 story 的 synergy（协同）**——我们不想让任何一方有短板。" 它最初是战术游戏，后来"变成更像视觉小说"，因为 VN 是更好的讲故事方式（但他们坚持两者都要有趣）。

### 1.3 公主连结 Re:Dive：动画级主线 + 约会式角色剧情，菜单上物理分离

公主连结（Cygames）以**动画品质的主线演出**著称（其改编 TV 动画也大获好评）。结构上：
- **主线（Main Story）**：在剧情菜单里，是一条完整的奇幻冒险线（失忆的男主 Yuuki + 美食公会一行），有反转、悬念、逐章升级的 stakes。叙事模式接近"公会 of the week"——核心小队稳定，每集/每段带 3-4 个新角色围绕一个主题登场。
- **角色剧情（Character Story）= 约会/social link 式**：一位评测者（Natalie）描述得很准确：这些是"**沉默、几乎不露面的主角**与某个女性可玩角色之间，通过某个活动或情境逐渐了解彼此的**约会式/社交链式互动**"。它们轻松、傻气、可爱，作用是"看着编剧和声优为美术给的设计注入生命"。
- **结构性洞察（极重要）**：公主连结把**故事和数值养成在 UI 上物理分离**——组队在角色菜单、关卡在任务菜单、抽卡在抽卡菜单、故事在故事菜单。这意味着玩家**很容易在前期被数值/体力/胜利的循环裹挟而跳过故事**；反过来也意味着"故事被设计成一个你想坐下来专门享受时再去看的独立区域"。

> 💡 **给新手的启示**：日式双层结构里，主线和角色剧情常常在玩法上是**两个独立入口**。角色剧情不是主线的支线，而是平行的、可选的、为亲密感服务的独立内容池。

### 1.4 双层结构总结表

| 游戏 | 主线层（plot） | 角色亲密层（intimacy） | 活动与主线关系 |
|---|---|---|---|
| **FGO** | 特异点/异闻带，7+章骨架，三幕中篇 | 羁绊 Lv1-10 → 资料/My Room 语音/幕间物语 | 活动是 if 时空，可让死者复活 |
| **蔚蓝档案** | Volume 制，"明→暗"曲线，日常即常态 | MomoTalk + 关系剧情，人人有聚光 | 活动是正史，与主线相连 |
| **公主连结** | 动画级主线，公会of the week | 约会式角色剧情，与沉默主角互动 | 活动剧情独立 |
| **赛马娘** | 育成 scenario（见下）+ 主线剧情模式 | 育成过程中的角色专属事件 | 季节/纪念活动 |

## 2. 角色第一的设计传统：原型 + 钩子 + 反差（ギャップ萌え）+ 声音

日式抽卡的角色，继承自**美少女游戏 / 约会模拟 / 偶像文化**。理解这个传统，新手就能用一套"配方"快速造出有吸引力的角色。

### 2.1 "Dere 谱系"——原型的元素周期表

日本动漫/galgame 文化沉淀出一套**可组合的性格原型词汇（"○○dere"）**，是角色设计的"乐高积木"：

| 原型 | 核心 | 一句话 |
|---|---|---|
| **Tsundere 傲娇** | tsuntsun（冷硬/暴躁）↔ deredere（热恋） | 对外凶/口是心非，对在意的人其实很在乎。又分 A 型（默认冷硬）/ B 型（默认甜，只对爱人凶）|
| **Kuudere 冷娇** | 表面冷静无表情、极少慌乱 | 外冷内热，骨子里很在乎 |
| **Dandere 默娇** | 害羞安静 | 遇到对的人才打开，变得温柔 |
| **Deredere 甜娇** | 内外都甜 | 不怕表达爱意 |
| **Yandere 病娇** | 起初友善 → 因爱黑化、偏执甚至暴力 | 爱让人疯狂 |
| **Bakadere 笨娇** | 表达爱时笨拙犯傻 | 但很深情 |
| **Oujodere 大小姐娇** | 端庄优雅、贵族做派 | lady-like |
| **Thugdere 不良娇** | 常以言行伤害他人包括爱人 | 用凶恶掩盖软弱的温柔面 |

> 这套词汇本身就是"原型 box"。但**只堆原型 = AI slop**。让角色活起来的，是下面的"反差"。

### 2.2 ギャップ萌え（Gap Moe / 反差萌）——角色架构的支柱

**Gap Moe = 角色的"外在表象"与"内在/行为"形成强烈反差**，这个反差本身就是吸引力。它被业界称为"专业角色架构的基础支柱"（ostaneskids 角色设计指南）。

**心理学机制（"Gain Effect / 增益效应"）**：为什么"凶角色做暖事"比"暖角色做暖事"更有吸引力？因为它**颠覆初始预期**。观众大脑看到两个相互冲突的特质，会**自动用自己的想象去填补缺失的部分**——动画导演鶴巻和哉直接把 moe 定义为"**自行填补角色缺失信息的行为**"。作者其实是在"骗"观众替他连点成线，观众一旦"参与"定义了这个角色，就会对他产生投入。

**Gap 的几种来源**（robynpaterson 总结，给新手当 checklist）：
- **行为反差**：在不同时间/地点/对象面前判若两人（对客人甜、对朋友是噩梦的不良少女）。
- **文化预期反差**：海盗本该是恶徒，但主角是友善的好海盗。
- **偏离自身原型**：讨厌见血的战士；被当成天才其实只是超级努力的人。

**经典 Gap Moe 配方示例**：
- 冷酷高冷的财阀继承人（kuudere）↔ 偷偷痴迷毛茸茸的猫（Persona 5 的 Jumin / Mystic Messenger）。
- 凶神恶煞的不良 ↔ 内心极度渴望被爱。
- 能征服王国的天才军师 ↔ 连煮个鸡蛋都不会（"能干的笨蛋"）。
- 严肃的学生会长 ↔ 一遇恋爱就脸红结巴（《魔王学院》Ameri）。
- 最成熟的成员 ↔ 真心相信圣诞老人（Love Live 西木野真姬，"成熟的孩子"）。

### 2.3 声优（CV/Seiyuu）的角色——不是配音，是"角色的灵魂"

在日式传统里，**声优不是辅助，而是角色身份的定义性力量**。研究与玩家调查反复指出：
- 大量玩家把"角色声音"列为选择玩某款游戏的**首要理由之一**。
- 2D 角色没有面部表情和肢体语言，**情感几乎全靠声优的表演承载**——音色、停顿、情绪起伏决定了角色如何被接收。
- 声音与外形必须"对上"：当外貌与音色对齐，角色才会显得"真实、像活人"。
- 声优是**跨媒体的**：游戏、动画、音乐、Live 演出连成一片，一个角色的声优本身就是营销资产（FGO、蔚蓝档案、偶像游戏都靠声优 Live 维系热度）。

> 💡 **给新手的启示**：即使你不实际找声优，也要在写角色时**为"声音"而写**——这个角色说话的语速、口头禅、语气词、第一人称（私/僕/俺/わたくし…中文里可类比"人家/老子/在下/本小姐"）。台词的"可被演出性"是日式角色设计的隐藏维度。蔚蓝档案/FGO 的角色资料里，"如何称呼自己""如何称呼你"往往是人设的第一笔。

### 2.4 如何**新鲜地**颠覆"原型 box"

原型是双刃剑：它给玩家快速的识别感，但用滥了就是套路。让原型新鲜的几条路（从上面材料提炼）：
1. **给原型加一个矛盾的 Gap**（第 2.2 节的核心）。
2. **让 Gap 与角色的"伤口/过去"挂钩**，使反差不只是搞笑而是有重量。蔚蓝档案 Wakamo 的反差之所以被称赞，是因为反差揭示了角色心理的深层变化，而不是单纯的笑点。
3. **用世界观给原型一个"为什么"**：蔚蓝档案每个学校对应一个文化原型（Abydos=埃及万神殿、Trinity=基督教三位一体、Gehenna=恶魔），角色的性格原型因此"长在世界观里"，而不是凭空的萌点。
4. **赛马娘式的"真实原型锚定"**：每个赛马娘的性格、剧情都基于**真实赛马的史实**（战法、宿敌、伤病、结局），原型因此有了不可替代的"真实重量"（详见第 6 节对照）。

## 3. 羁绊/绊（絆）剧情 & 关系叙事：一篇"好的角色小剧情"长什么样

短角色小剧情（vignette）是日式抽卡建立依恋的主力。它们的"养成最佳实践"祖师爷是 **Persona 的 Social Link / Confidant（社交链/密友）系统**。

### 3.1 Social Link：10 段式的羁绊模板（最该抄的结构）

Persona 的社交链是抽卡羁绊剧情的直接祖先，结构极其清晰：
- **每条社交链分 10 个 Rank（等级）**，每个 Rank 对应一个**独立的小事件**。
- 两大目的（mchllshell 分析）：**技能成长（skill-building）+ 世界塑造（world-building）**。即羁绊剧情既给玩法增益，又顺便扩展世界与人物。
- 每个角色对应一张塔罗牌（Arcana）。**升满一条链解锁该角色的"终极人格/真结局"**。
- 互动机制：选对话选项 → 加好感（冒出金色音符）；带对应 Arcana 的 persona 能加速。

**一条好的社交链的内部弧线**（从 Persona 反面教材里也能学）：
- 一条被批评的链（Persona 5 Makoto）"感觉到处乱跳"、"Makoto 在自己的社交链里反而像配角"、恋爱来得太突兀——**反面教训：羁绊剧情必须始终以这个角色自己的成长为主角，情感推进要有铺垫节奏，不能突然降临。**
- 一条被称赞的设计（女主限定的 Shinjiro 链）甚至能**反过来小幅影响主线事件**——羁绊层与主线层产生交互，是高阶玩法。

### 3.2 蔚蓝档案 MomoTalk：IM 体的关系剧情

- 形态：**手机聊天 App**，玩家（Sensei）和学生一对一聊天，穿插选项。
- 节奏：碎片化、随时可读，作为主线之间的"插曲"。
- 价值：保证**每个角色都有专属聚光**；通过日常对话累积熟悉度与依恋。

### 3.3 赛马娘育成 scenario：把"羁绊"做成一整段角色弧线（高阶范例）

赛马娘的"育成模式（Career Mode）"是一个**roguelike 式的、把单角色故事压缩进一次 run 的精妙设计**：
- 一次 run ≈ 3 年（72 回合 + 6 回合 finale），每月分两个 turn。玩家在训练/外出/休息/比赛之间做选择。
- **关键剧情事件在固定时间点发生**（推进该 scenario 或该角色的故事），其他事件随机触发。
- 演出极其讲究："镜头角度、丰富的动作，角色进出场景、与物体互动，镜头跟着场景移动——**几乎像在看一部剧集**"（gamerefinery 拆解）。

**一个具体的角色弧线案例——サイレンススズカ（Silence Suzuka）的育成剧情**（基于一篇详尽的日文逐事件分析 note.com/mutsukomtk）：
- 史实锚点：Silence Suzuka 是一匹以"大逃げ（大幅领跑）"著称、却在天皇赏·秋的比赛中因伤倒下、被安乐死的悲剧名马。
- 育成剧情把一条**存在主义的角色提问**编织进固定的比赛事件序列里："我跑步到底是为了什么？""赢，就意味着踩在败者梦想的尸体上——我有资格粉碎别人的梦吗？"（"胜者站在败者梦想的尸骸之上"）。
- 这条心理线索贯穿弥生赏→夏合宿→毎日王冠→天皇赏（秋）等真实赛程节点。每个比赛结果（第几名）触发不同的剧情反应（如"取り戻したもの/找回的东西"、"そして光へ/然后，向着光"），把数值结果和情感节拍绑在一起。
- 通过让后辈（エルコンドルパサー等）把 Suzuka 当成"要超越的目标"，反过来让 Suzuka 第一次意识到自己跑步的意义——**用配角的视角来照亮主角的内心**，这是 Uma Musume 叙事的招牌手法（评论指出第一季里 Special Week 其实不是推动剧情的人，剧情由 Silence Suzuka 推动，Special Week 靠"与 Suzuka 的友情"被系到叙事里）。

> 💡 赛马娘的范例告诉新手：**一段角色小剧情也可以有完整的"提问—挣扎—顿悟"弧线**，而不只是甜蜜日常。把"角色要面对的一个内在问题"作为骨架，让剧情事件逐个推进这个问题，结尾给一个情感顿悟。

### 3.4 偶像游戏的 produce 模式：一对一关系叙事

偶像大师系列（特别是**学园偶像大师 Gakuen iDOLM@STER / Gakumas**，QualiArts/Bandai Namco）把羁绊做成"制作人养成偶像"：
- 玩家是"制作人（Producer）"，一对一负责培养一名偶像，帮她度过比赛、克服自己的问题、揭示真正的潜力。
- 官方对叙事的定位极具启发性：
  > "**作为偶像的成长，会通向作为'人'的成长。**随着 produce 的推进，你和偶像之间一对一的故事会展开——她们作为偶像的弱点、隐藏的烦恼……作为制作人，与女孩们紧密合作，加深羁绊，一起成为顶级偶像。"
- 结构上同样借鉴了 Uma Musume 的"训练 roguelike"（4 季 × 8 周 = 一次 produce run，每周一个行动）。每个偶像有"主线交流（Hatsuboshi Commu）+ 好感交流（Affection Commu）+ 卡面交流（Card Commu）"三层。

> 💡 偶像游戏给出了"羁绊剧情"的另一个母题模板：**"成长为偶像 = 成长为人"**——用一个外部目标（成为顶级偶像/赢得比赛）当外壳，内里讲的是这个角色克服自己心理弱点、变成更完整的人。这是把"养成数值循环"和"角色弧线"焊在一起的经典手法。

### 3.5 "好的羁绊/角色小剧情"通用模板（提炼）

```
【一篇好的角色羁绊小剧情模板】（适合 3-10 段的 social-link 式结构）

钩子（Rank 1）：呈现角色的"表象"——他的原型、口头禅、对你的初始距离感。
              埋下一个"裂缝"：一个不对劲的细节，暗示表象之下另有东西。

挣扎（中段）：通过一个共同活动/情境，裂缝逐渐展开。
            角色面对一个属于他自己的"内在问题/伤口/弱点"。
            ⚠️ 这个角色必须是他自己故事的主角，"你"是催化剂不是主角。
            情感推进要有铺垫，不能突然降临（Persona Makoto 反面教训）。

顿悟（后段/满级）：角色在与你的关系中获得对那个内在问题的回答/释怀/成长。
                Gap 在此被揭示出有"重量"（不只是搞笑反差，而是心理真相）。
                给玩家一个"只属于你和他"的私人时刻（独占感）。

收束：解锁"终极形态"——更深的资料、隐藏语音、真结局，或反哺主线的小钩子。
```

## 4. 活动 / 季节叙事：if/AU 传统与"无主线重量的聚光灯"

日式抽卡有一套成熟的**轮换季节活动**传统，几乎所有抽卡都做（夏日/泳装、圣诞、万圣、新年、情人节、联动 crossover）。

### 4.1 两种活动哲学：if 时空 vs 正史

- **FGO 式 = if/AU（平行/架空）时空**：夏日活动是"自成一体的小现实"，从者的 Spirit Origin（灵基）可以随意更换（变成泳装泳衣版本），活动里发生的事**不影响主线正史**。Nasu 还点明 FGO 刻意用**真实世界节日**（万圣、圣诞）而不是像 HoYoverse 那样原创节日。活动是作者的自由游乐场——可以让主线里死掉的角色复活、可以玩崩主线里的严肃设定。
- **蔚蓝档案式 = 正史活动**：活动是发生在主线 Volume 之间的 canon，与主线相连，会被主线回忆引用，强化"日常即常态"的整体氛围。

> 💡 这是新手要做的一个明确选择：**你的活动是"番外搞笑特别篇"（if），还是"正史的日常切片"（canon）？** 两种都成立，但要一致。混着来会让玩家困惑"这到底算不算数"。

### 4.2 季节活动的功能

- **给心爱角色一次无主线重量的聚光**：泳装/和服/圣诞版本让一个角色以新形象、新情境重新登场，不必背负推进主线的责任。
- **变体即新卡**：同一角色的泳装版本就是一张新的可抽卡（蔚蓝档案 Hoshino (Swimsuit)、Miyako (Swimsuit)…；FGO 泳装从者）。一个角色因此可以被"反复变现"。
- **联动/crossover**：跨 IP 活动既吸引对方粉丝（拉新），又用独占内容取悦老玩家。FGO 甚至联动自家《空之境界》，但 Nasu 刻意**不触及原作主线剧情**，让没读过的人也能当"有趣的新从者"享受，读过的人则会心一笑——**这是处理联动剧情的黄金法则：对新人是独立好故事，对老粉是彩蛋。**

### 4.3 活动/AU 模板（提炼）

```
【季节/AU 活动模板】

1. 选一个"情境外壳"：夏日海滩 / 圣诞 / 文化祭 / 温泉 / 联动世界。
2. 决定 canon 性：if 时空（可玩崩、可复活死者）or 正史日常切片（与主线相连）。
3. 选 2-4 个聚光角色（蔚蓝档案规则：一次最多放 2 个新角色，便于聚焦；
   碧蓝航线一次放 4-6 个则更难塞进集中的剧情——少即是聚焦）。
4. 给他们一个"脱离主线重量的轻情境冲突"：一场比赛、一次误会、一个小任务。
5. 用情境揭示角色平时主线里看不到的一面（泳装版本 = 性格的新切面）。
6. 联动法则：对新人是独立好故事，对老粉是彩蛋；不强迫玩家懂原作。
```

> 注意（碧蓝航线的反面教训）：碧蓝航线一次引入 4-6 个新角色，导致"活动很难把老角色塞进既有剧情线"、需要少数角色在说明性场景里"无处不在"。**一次聚光的角色越少，剧情越能聚焦。**

## 5. "剧情驱动 vs 玩法驱动"光谱：故事型抽卡为何成立

抽卡游戏在"故事是不是主要卖点"上是一条光谱：

| 一端：故事是卖点 | 中间 | 另一端：玩法/收集为主 |
|---|---|---|
| **FGO**（故事是核心驱动力，Nasu 级主线）| 公主连结（动画级主线但故事在独立菜单，易被跳过）| 碧蓝航线（剧情较轻，收集/养成为主）|
| **蔚蓝档案**（"让人留下来的是故事"）| 赛马娘（玩法 roguelike + 强情感 scenario）| 多数轻剧情抽卡 |
| **NIKKE**（电影化、故事重）| 偶像游戏（produce 玩法 + 一对一叙事）| |

### 5.1 NIKKE：把"剧情第一"做成开发哲学

NIKKE（韩国 SHIFT UP，导演 Yoo HyungSuk 유형석）是"故事重型"抽卡的代表，导演访谈给出明确哲学：
> "我认为游戏里**最重要的就是 scenario（剧本）**，因为我相信一切都从剧本开始。"
> 他在上线后调整了优先级——从追求技术卓越，转向"**对用户的爱才是最重要的……开发方和用户之间的情感连接、用户爱这个游戏，才是最重要的**"。
> 灵感来自《英雄传说 III：白发魔女》——"我学到了剧本和音乐能多大程度影响一款游戏。"
- **长线叙事规划**：NIKKE"未来两年的主线大纲已经规划好了"。2026 年的 Developer's Note 显示其叙事是**主线更新 + Side Story（侧线）更新交替**的节奏（2 月主线、4 月侧线/主线、8 月主线、10 月侧线/主线），并强调"确保角色、战斗、动画、音乐、叙事元素相互配合，创造更有凝聚力的体验"。
- 导演的野心是让 NIKKE"成为一款能持续 20 年以上的游戏"——靠强叙事地基 + 灵活内容适配。

> 💡 故事型抽卡成立的共同要素（从 FGO/蔚蓝/NIKKE 提炼）：**(1) 故事被当成"让人留下来"的核心引擎而非附属品；(2) 有一个可数的、有终点感的主线骨架；(3) 主线 + 角色亲密层双轨并行；(4) 长线大纲先行（两年起规划）；(5) ADV/VN 式演出让文本可大量、低成本产出。**

### 5.2 视觉小说 / ADV 演出血脉

故事型抽卡的呈现方式直接继承自**日本视觉小说 / ADV（adventure game）传统**：
- **谱系**：1983《港口连续杀人事件》（Portopia，堀井雄二/Enix）开创"command select"式 ADV → 分两支：一支通向《逆转裁判》《弹丸论破》（推理/指令式），另一支是 90 年代的 **bishōjo（美少女）novel game**（Leaf 1996《雫》被视为第一部真正的"视觉小说"；Leaf《To Heart》、TYPE-MOON 2000《月姬》）。约会模拟更早（1987 Famicom《中山美穂のトキメキハイスクール》）。
- **演出语法**：静态角色站立绘 + 背景美术 + 对话框 + 配乐/音效 + 菜单式选项。叙事和互动都通过文本框呈现，选项控制角色言行或心理。
- **为什么抽卡爱用它**：**廉价、可规模化**。画几张差分立绘 + 写文本 + 配音，就能产出海量剧情。这正是 FGO 每月能产 25 万字、蔚蓝档案能给每个角色配 MomoTalk 的根本原因。
- TYPE-MOON 出身就是同人 VN（《月姬》），所以 FGO 的"对话剧场"本质是把 VN 演出搬进抽卡。

> 💡 **给新手的启示**：日式抽卡的剧情写作，本质上是在写**视觉小说脚本**——你的"摄影机"就是立绘的进出、表情差分、对话框、和配乐提示。学会用"立绘 + 对话框"这套极简语法讲故事，是日式路线的基本功，也是成本最低的起点。

## 6. 与中国/米哈游模式的对照：两种传统的心智模型

这是给新手最重要的决策框架。**两套传统在"主菜""演出""角色锚点""更新结构"上系统性不同。**

### 6.1 核心对照表

| 维度 | 日式传统（FGO/蔚蓝/公连/赛马娘/偶像）| 中式/米哈游传统（原神/星铁/绝区零）|
|---|---|---|
| **"主菜"是什么** | **角色**第一；世界是让角色发光的容器 | **世界 + 主线**是旗舰；角色镶嵌进宏大世界 |
| **叙事结构** | 中央主线 + 大量单角色羁绊/好感小剧情**双层并行**，玩法上常物理分离 | 世界主线为骨干 + 角色传说任务/邀约事件作为亲密层（后期补上的）|
| **演出形态** | **ADV/视觉小说式**：立绘 + 对话框 + 配音，廉价可规模化 | **电影化**：3D 场景、全程语音、过场动画、交响乐（伦敦爱乐/上海交响/东京爱乐）|
| **角色锚点** | 锚在**原型 + 反差萌 + 声优**；或锚在真实史实（赛马娘=真实赛马，FGO=神话英雄）| 锚在**世界观/地区文化**（蒙德=西方自由、璃月=古中国契约、枫丹=工业革命欧洲），角色是原创世界的"原生公民" |
| **更新单位** | Volume / 特异点 / 育成 scenario / 季节活动 | 大版本（地区）+ 角色传说任务 + 原创节日活动 |
| **节日/活动** | 多用**真实世界节日**（万圣/圣诞/夏日泳装），常为 if 时空 | 多**原创节日**（海灯节等），嵌进世界正史 |
| **成本/规模** | 文本驱动，单位成本低，月产可达数十万字 | 工业化、超高预算（原神开发+营销约 1 亿美元）|
| **典型玩家黏性来源** | "我和这个角色的私人羁绊" + 角色 IP/声优 Live | "探索这个世界" + 角色在世界里的故事 + 沉浸式视听 |

### 6.2 学术与业界对照证据

- **角色锚点的根本差异**（davidpublisher 跨文化研究，对比原神 vs 赛马娘）：原神角色基于**原创世界的地区文化**设计（蒙德西式自由、璃月中式契约），靠"独特原创角色"积累粉丝；赛马娘角色全部基于**真实赛马**，靠"史实人格化 + 偶像/赛马元素结合"积累粉丝，史实让"故事更真实、更能引起日本玩家共鸣"。→ **日式倾向于在'已有的真实/神话素材'上做人格化；中式倾向于'从零创造一个自洽世界'。**
- **角色故事 = 留存与回归的关键**（diva-portal 研究）：即使在米哈游模式里，玩家也明确指出"角色故事"是他们回归游戏的理由——每出一个新角色就配一个角色故事，让玩家熟悉并产生情感依恋。→ **两套传统最终殊途同归：角色亲密层在哪种传统里都是留存核心。区别只在'它是第一性的还是后补的'。**
- **Nasu 自己点出的对照**：他明确说 FGO 用真实节日而非"像 HoYoverse 那样创造原创节日"。这是创作者亲口承认的两套路线分野。
- **工业规模差异**：日本开发者自己承认"在中国游戏工作过就知道，制作规模是另一个量级"（automaton-media）。中式旗舰是用工业化电影演出在"抬高门槛"，日式则用文本/ADV 的轻成本换取"角色数量与更新频率"。

### 6.3 给新手的"选边或混搭"决策

```
【选哪套传统？——三个问题】

Q1. 你的"主菜"是角色，还是世界？
    → 角色为主 ⇒ 日式：先把角色 box（原型+反差+声音）做扎实，
                  主线只是让他们轮流发光的舞台。
    → 世界为主 ⇒ 中式：先把世界观/地区/规则做自洽，
                  角色是这个世界的原生公民。

Q2. 你的演出预算 / 产能是什么量级？
    → 低成本 / 高频更新 ⇒ 日式 ADV：立绘+对话框+配音，
                          学会用极简语法讲故事。
    → 高预算 / 追求沉浸 ⇒ 中式电影化（但新手几乎不可能单干，
                          先用日式语法把故事讲对再谈演出）。

Q3. 你的角色锚在哪？
    → 锚在"真实素材/神话/历史"（赛马、英灵、名将）⇒ 日式人格化路线，
       借真实重量。
    → 锚在"原创世界的文化设定"⇒ 中式原生路线，
       但要先建好世界这个"母体"。

【混搭建议（当代主流）】
- 用中式的"世界主线为骨干 + 电影化关键节点"做旗舰感，
- 用日式的"单角色羁绊/好感双层 + ADV 日常小剧情"做留存与亲密感，
- 两层并行、但要明确哪层是你的"第一性"。
  蔚蓝档案（日式底子 + 越来越史诗的主线）和米哈游（中式底子 + 后补角色传说任务）
  正在从两端向中间靠拢——但它们各自的"重心"仍清晰可辨。
```

## 7. 四个可复用模板（汇总，给新手直接套用）

### 7.1 主线 + 角色 双层规划模板（Main + Character Two-Layer Plan）

```
【第一步：画"空宝箱"（Nasu 法）】
- 整部故事 = N 个可数的大块（特异点/Volume/章），每块写一句"要完成什么"。
- 决定整部的"终点感"：玩家从第一天就知道目标（"清掉这 7 章"）。
- 先想清楚整部 plot，再把每块"该做什么 + 钩子"作为材料下发/自我约束。

【第二步：定主线层的曲线】
- 蔚蓝档案曲线：明亮日常起 → 逐渐变深刻/严肃（"日常即要守护的常态"）。
- 每块用三幕：引入"有什么不对劲" → 边走边搞清 → 揭示+高潮（FGO 章节法）。
- 设一条铁律管理"代价"（如：主线死亡的角色不再回主线，强化重量）。

【第三步：搭角色亲密层（与主线物理分离的独立入口）】
- 每个角色一条羁绊线（10 段式 social-link / MomoTalk / 育成 scenario 任选）。
- 保证人人有聚光，不只主推角色。
- 亲密层的事件 = "你和他的私人关系"，主角始终是那个角色自己。

【第四步：连结两层】
- 决定活动是 if 时空还是正史；正史活动要与主线回忆相连。
- 高阶：让某些羁绊进度小幅反哺主线（Persona Shinjiro 法）。
```

### 7.2 角色钩子配方（Archetype + Gap Recipe）

```
【一句话角色配方】
[一个识别度高的原型/dere] + [一个矛盾的反差(Gap)] + [一个可被演出的"声音"] + [一个内在伤口/问题]

示例：
- 原型：高冷大小姐（oujodere/kuudere）
- 反差：私下偷偷沉迷地摊小吃 / 怕黑 / 连煮蛋都不会
- 声音：自称"本小姐"，句尾爱加"哼"，对你用敬称却口是心非
- 伤口：童年被当成"完美继承人"养大，从没被允许"普通"过
       → 这个伤口让"怕黑/笨拙"的反差有了重量，而不只是搞笑

【新鲜化检查（避免 AI slop）】
□ Gap 是否与角色的过去/伤口挂钩，而非纯笑点？
□ 原型是否"长在世界观里"（有一个'为什么是这样'）？
□ 这个角色说话时，我能否"听见"他独特的语速/口头禅/第一人称？
□ 我是否在为"声优能怎么演"而写台词？
```

### 7.3 羁绊/角色小剧情模板（Bond Episode Template）

见 §3.5。核心三拍：**钩子(露表象+埋裂缝) → 挣扎(裂缝展开，角色面对自己的内在问题) → 顿悟(在与你的关系中获得成长，Gap 显出重量，给独占时刻)**。角色始终是自己故事的主角，"你"是催化剂。情感推进必须有铺垫（Persona Makoto 反面教训）。

### 7.4 季节/AU 活动模板（Event/AU Template）

见 §4.3。核心：**情境外壳 → 定 canon 性(if 还是正史) → 少量聚光角色(≤2-4) → 脱离主线重量的轻冲突 → 揭示角色新切面 → 联动法则(对新人独立好故事，对老粉彩蛋)**。

## 8. 不确定性与边界声明（诚实标注）

- **FGO 章节细节**：第一部 = Singularity F + 第一~第七 + 最终特异点 + Solomon（这点多源一致：百度百科 Singularity 词条、fgo.gamepress、rpgsite）。但各章节的具体剧情/角色，我只引用了 Avalon le Fae 的"三幕结构"这一被多篇评论印证的结构性判断，未编造具体情节。
- **蔚蓝档案主笔变更**：原主笔 isakusan（양주영/Yang Joo-young）与部分核心成员 2024 年离队成立 Dynamis One，主笔换为 Oh Hyun-seok；后续有 Project KV 抄袭争议与 Nexon 报警调查（automaton-media 报道）。本文引用的叙事哲学来自更早的 Yong-Ha Kim / isakusan 时期访谈，代表的是蔚蓝档案"经典期"的设计思想。新手参考其方法论无碍，但需知道团队已变动。
- **Silence Suzuka 育成剧情**：基于一篇详尽的日文逐事件分析（note.com/mutsukomtk）与攻略站（gamewith/kamigame）的事件名，情感解读为该作者的分析，非官方文本逐字引用。史实（大逃げ名马、天皇赏·秋倒下）为公认事实。
- **"两种传统"心智模型**：这是一个**教学用的简化框架**，真实世界两端在融合（已在 §0、§6.3 反复声明）。不要把它当成铁律，而是当成新手起手时的"重心定位器"。
- **NIKKE / 偶像游戏访谈**：NIKKE 导演 Yoo HyungSuk 的引语来自 Saiga NAK 访谈与官方 Developer's Note；偶像游戏叙事定位引自官方 Google Play 描述与 wiki，均为可核查的一手/官方来源。
- 未能深入的盲区：碧蓝航线的具体剧情写作（资料多被反爬墙挡住，只取到"一次引入 4-6 角色导致剧情难聚焦"这一结构性观察）；韩系抽卡（蔚蓝档案、NIKKE）的韩文一手创作访谈（语言与可及性限制，主要依赖英译访谈）。
