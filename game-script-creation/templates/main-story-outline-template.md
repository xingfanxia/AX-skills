# 主线框架表（Main Story Outline）

> **怎么用**：把"要讲什么"排成有结构的骨架。先选一个**结构框架**（按故事形状），再填它的关键转折点——不是每一场戏，只是骨架。
> （框架详见 `references/01-story-architecture.md` §2-4；二游长线主线模板见 `references/05` §10.A；"空宝箱"法与主线/羁绊双层规划见 `references/06` §1.1 与 §7.1）

---

## 第一步：选框架（按故事形状，别一套套到底）

| 故事形状 | 推荐框架 |
|---|---|
| 标准单章 / 单弧 | 三幕 或 救猫咪 15 拍 |
| 史诗 / 被选中者 RPG | 英雄之旅（Vogler 12 阶）|
| 角色向小剧情 / 日常 | 故事圈（8 步）或 起承转合（不需驱动冲突）|
| 紧凑悬疑 / 推理弧 | 七点式（从结局倒推）|
| 长线连载（二游/MMO）| 分形三幕（宏观+中观+微观）|

> **我选的框架**：____________

---

## 第二步：最小骨架（7 锚点，框架无关，先填这个）

| 位置 | 要回答的问题 | 我的填写 |
|---|---|---|
| ~10% | 激励事件是什么？（世界被打破的那一下）| |
| ~20% | 主角做了什么决定，从而被卷入故事、无法回头？ | |
| ~50% | **中点翻转**：发生了什么让局面与之前完全不同？（假胜/假败/重大揭示，且主角从被动转主动）| |
| ~75% | 最低谷是什么？他失去了什么？ | |
| ~80% | 什么领悟/资源让最后一搏成为可能？ | |
| ~95% | 高潮（最终对决）是什么？ | |
| ~100% | 故事结束后，世界变成什么样？ | |

> **节拍连接自检**：相邻节拍之间是用"但是(复杂化)/所以(后果)"相连的吗？凡是只能用"然后…然后…"连起来的，就是没有因果，要改。

---

## 第三步（二游长线必填）：分章 + 伏笔账本 + 双重收尾

### 分章 / 分幕（地区=幕 或 大章套小章）
| 篇章 | 母题（拷问哪个当代真实情绪）| 当前问题（本章解决，给闭合感）| 更高层留坑（给追更动力）|
|---|---|---|---|
| 第一篇章 | | | |
| 第二篇章 | | | |

### 空宝箱定终点（Nasu 法）
> 整部故事 = ___ 个可数的大块；玩家第一天就知道的"总目标"是：____________

### 伏笔账本（只有"总编"能改）
| 伏笔 / 钩子 | 埋于 | 计划在哪一章/版本回收 | 状态 |
|---|---|---|---|
| | | | |

> **每个篇章双重收尾铁律**：①闭合一个当前问题（情绪满足）+ ②打开一个更大问题（追更动力）。两者缺一不可。

## 交付前自检
- [ ] 选定了框架并填满 7 锚点
- [ ] 有一个真正的中点翻转（不是中段注水）
- [ ] 节拍之间是"但是/所以"，不是"然后/然后"
- [ ] （二游）每个篇章都有"当前收束 + 更高层留坑"
- [ ] （二游）伏笔账本里每个坑都标了回收章节
