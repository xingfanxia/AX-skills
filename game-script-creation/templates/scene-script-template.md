# 场景脚本模板（Scene Script）

> **怎么用**：把框架里的一个节拍写成一场能演的戏。先用「A 场景规划」想清楚目标与灾难，再写正文。
> - **线性场景**（二游 ADV / 视觉小说 / 剧情向）→ 用 **B（ADV 演出脚本）** 或 **C（对白表）**。
> - **非线性 / 系统化对白**（Roguelike / 模拟 / 沙盒 / 竞技）→ 用 **D（系统化对白池）**。
> **填写区在 `→` 或代码块占位处；「示范」用的是另一个题材（潮汐邮局），请勿照抄。**
> （craft 详见 `references/07` Part A；格式/演出/工具/本地化详见 `references/08-script-formats.md`；非线性叙事见 `references/04` §6.5–6.7）

---

## A. 场景规划（写之前先想清楚）
- **POV 角色**（一场一个视角，别跳头）→ 〔填写〕
- **场景目标（Goal）**：这个角色进场时**具体**想要什么？→ 〔填写〕
- **冲突（Conflict）**：什么挡着他？→ 〔填写〕
- **灾难（Disaster）**：这场戏**不**干净地解决——哪种？ [ ] 直接的"不" [ ] "是的，但是…" [ ] "不，而且更糟"
- **续景（反应→两难→决定）**：他做的决定会变成下一场戏的目标 → 〔填写〕
- **这场戏要让读者感受到/相信什么**（写给读者的目标，不是倒空脑里的信息）→ 〔填写〕

---

## B. ADV 演出脚本（二游 / 视觉小说 · 线性场景）
> 演出标记用 `[方括号]`。**表情用中文 + 英文键**（如 `戒备(guarded)`），英文键是给美术/引擎的资产名，开写前与美术约定；只用资产表上已有的表情。关键情绪点要"文本 + 演出"合力，但警惕静态对话墙。

```
========================================
场景名称: 〔填写〕   场景编号: SCN_____
========================================
[背景: bg_____ | 切换: 淡入/切 | 时长: ]
[BGM: BGM_____ | 音量: % | 淡入: s]

--- 旁白 ---
（环境/动作描写。用起作用的感官细节锚住场景，避免"白屋症"。）

--- 角色名 ---
[立绘: ____ | 位置: 左/中/右 | 出现: 淡入/滑入 | 表情: 中文(英文键)]
"台词。"           ← 对白要有潜台词；删"他看见/她感到"这类滤镜词
[配音: char_scn_行号]
[演出FX: 屏幕震动/闪白/CG插入 | 强度 | 时长]   ← 只在关键情绪点用

--- 选择支 ---
[UI: 显示选项框]
  选项 A: "____"  [配音: ____]  → 跳至: SCN_____A  [SET FLAG: ____]
  选项 B: "____"  [配音: ____]  → 跳至: SCN_____B  [SET FLAG: ____]

--- 汇合点 (SCN_____CONVERGE) ---
（每个分支都必须有 → 出口，不能死路。）
[背景: 淡出][立绘: 淡出][BGM: 淡出]
--- 场景结束 ---
```

## C. 对白表（需要分支/本地化/配音时用，多数手游主力格式）
| line_id | scene_id | speaker | 表情 | zh_text | voice_file | branch_to | condition | notes |
|---|---|---|---|---|---|---|---|---|
| 〔填写〕 | | | | | | | | |

> `line_id`/`voice_file` 一旦进入配音/本地化就**冻结**，不可改名。`zh_text` 与译文分列，不互相覆盖。单句中文超过 25 字要警惕本地化爆框。

## D. 系统化对白池（Roguelike / 模拟 / 沙盒 / 竞技 · 非线性叙事）
> 这类游戏不写线性场景，而是写**按状态触发的对白池**：同一个交互点，按上下文给 5–10 条变体；把"死亡/重开/重逢"当叙事递送时机（Hades 法）。
```
交互点: 〔如"每次重开后在营火遇见 NPC_老兵"〕
触发变量: run_count / last_death_boss / flag_xxx / 好感等级 ...
对白池:
  [条件: run_count == 1]            → "第一次？……活着回来再跟我吹。"
  [条件: run_count >= 5]            → "又是你。这次别死在第二层了。"
  [条件: last_death_boss == 'X']    → "被 X 收拾了吧？我看你的脸就知道。"
  [条件: flag_救过老兵 == true]      → "上次那事……我记着。"（解锁专属支线钩子）
  ...（高频交互至少 5–10 条变体，避免重复听腻）
设计要点: 早期台词在玩家通关全貌后要能"重听出新意"；变体覆盖首次/多次/刚失败/旗标已置位。
```

## 📝 填写示范（题材：潮汐邮局 · 角色"闻汐" · 请勿照抄）
```
场景名称: 退潮前的那封信   场景编号: SCN_TIDE_01
[背景: bg_灯塔邮局_黄昏 | 淡入][BGM: BGM_退潮 | 50% | 淡入 2s]
--- 旁白 ---
潮水正在退。沙滩上，几封没投出的信开始发软、边缘化进水里。
--- 闻汐 ---
[立绘: 闻汐·制服 | 中 | 淡入 | 表情: 平静(neutral)]
"还有两刻退潮。手里的，都投得出去。"
[配音: wenxi_tide01_010]
[立绘: 闻汐 | 表情: 戒备(guarded)]   （她的手按住了胸口口袋——那里有封没编号的信。）
"……除了不该我投的那封。"
[配音: wenxi_tide01_020]
[演出FX: 海浪声渐强 | 0.4s]
```
（点评：开场没甩"潮汐/邮局"设定名词；"还有两刻退潮"既是报时也是核心特质；按住口袋是潜台词，也是灾难前的决定种子。）

## 交付前自检（结合反 AI 味检查 `references/09-anti-ai-slop.md`）
- [ ] 这场戏有**目标 + 灾难**；续景有两难+决定
- [ ] 对白有**潜台词**，无"你知道的，鲍勃…"式信息倾倒
- [ ] 盖住名字能认出谁在说话（角色声音可区分）
- [ ] 删掉了滤镜词（看见/听到/感到/似乎）；有起作用的感官细节
- [ ] 一场一个 POV，没跳头
- [ ] （线性/二游）关键情绪点标了演出；不是静态对话墙；表情都在资产表上
- [ ] （非线性）高频交互有足够变体；死亡/重开被当成叙事时机
- [ ] 每个分支都有出口
