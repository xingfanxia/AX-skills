## 三、这个 skill 能帮用户做什么

**一条全流程陪伴线**，每步交付一份实体文档，最终汇成「创作圣经」：

```
S0 校准 → S1 种子定调 → S2 世界观 → S3 核心理念(主题)
        → S4 主线框架 → S5 人物核心特质 → S6 具体场景脚本 → S7 修改
```

**对应全部需求的关键特性：**

1. **从模糊到具体的 full loop**：把"脑子里一个画面/感觉"一路引导到世界观、主题、主线框架、人物特质，再**辅助写出具体主线与场景对白脚本**（不止做设定，能真正落笔成戏）。
2. **自适应水平**：开场用小样本判断真实水平（零基础 / 有想法没技巧 / 有直觉没训练 / 已能成稿）调旋钮——不把人当纯小白淹没，也不居高临下。**v2 增加"工作模式选择器"（A–E）**：让创作者主动选 AI 介入多深（只给思路 ↔ 直接给初稿），与水平校准正交。
3. **中文为主、多语言支持**：全程中文交流/提问/示范/产出；术语先中文再括注英文。
4. **通用适配所有游戏类型，二游覆盖最深**：RPG/开放世界/视觉小说/Roguelike/模拟/解谜/竞技/沙盒/4X/恐怖/MMO/卡牌都能走；二次元抽卡有最深专项。
5. **（v2 新增）把场景写到"扎心/好笑/心动/想追更"**：四种情感工艺（刀/笑/萌·恋/谜）让每一拍情感真正落地，而不只是骨架。
6. **（v2 新增）把剧情做成能持续供货的内容机器**：可抽角色价值栈、卡池-版本协同、玩家身份模型、可跳过写作、长线版本路线图、篇幅基准——从"写一个故事"升级到"运营一个长线二游项目"。

**怎么用**：直接说"我想做个游戏剧情但不知从哪开始""帮我写一场催泪/搞笑的戏""做二次元抽卡手游想写主线/角色/活动剧情/卡池"即可触发。拿不准某步产出长什么样，看 `examples/worked-example-erciyuan.md`。

## 四、质量保障

- **v1**：8 个调研流并行（二游 deep dive ×2 与写作/辅导流用 Opus；二游与日韩/写作流用 Tavily）；成稿后跑 2 个独立 Opus 评审，发现问题全部修复复验。
- **v2**：
  - 整合三份外部产出（两份 compass 深研究 + 一份完整中文 skill 草案）。
  - 二轮深度调研全部 **Tavily-backed、force web search、一手来源、⚠️ 诚实标注**：Wave 1（绝区零/鸣潮补盲 + 2025-26 时效 + 引述核查与概念纠正），Wave 2（刀/笑/萌恋/谜四种情感工艺 + Emily Short 选择架构 + 序章 onboarding）。
  - **修正一个概念框架**：character-first vs world-first 这条轴**正交于国别**——纠正了旧的"中式=世界第一"说法（米哈游其实是角色第一的电影化包装；真正世界第一的中国工作室是鹰角/深蓝、韩国是 Project Moon）。
  - 成稿后跑 **2 轮独立 Opus 评审**（可用性 GREEN 测试 + 一致性/准确性/反 AI 味审计），发现的全部问题（旧框架在 creative-bible 模板的残留、一处自违反 anti-slop "抓手"、价值栈示范不完整）**全部修复并复验**；全量 grep 确认无框架残留、无 banned-slop、12 篇 reference 交叉引用全部解析。
