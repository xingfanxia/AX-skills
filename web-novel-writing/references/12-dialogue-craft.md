# 12 · 对话工艺（网文对白）

> Purpose: 把网文对白从"两个角色轮流播报信息"的 AI 默认态，改造成可约束、可审、部分可机检的工程对象——用**长度差编码权力**、用**议程碰撞造张力**、用**7 维差异化让每个角色有自己的声音**。**用在步骤4（writer 写对白）和步骤6（reviewer 审 `dialogue_natural` 子判据）。** 本文不另起体系，全部挂在已有契约之上：人物卡 `voice_notes`/`cognition`/`behavior_anchors`（`templates/state-characters.yaml`）、爽点排布 `sao_schedule`/打脸四部曲（`references/07`）、反 AI 味 Gate E/G（`references/09`）、审校项 `dialogue_natural`(max15/min9, `references/04`)。

---

## 0. 一句话立场

对话是网文里**最容易写出 AI 味**的体裁——AI 默认把对话写成"一问一答的审讯稿 + 每个角色一个腔调 + 用台词整段科普设定"。本 skill 不靠"写得自然"这种玄学，而是把对白拆成**四条可下指令、可审、部分可机检的规则**：① 对话长度=权力地位；② 每个角色带自己的议程进场；③ 7 维差异化让"遮住名字也认得出是谁"；④ 角色不当科普嘴。前三条是"怎么写好"，第四条是"绝不能犯"（并入 info-dump 裁决）。和爽点引擎（`07`）一样：**对白质量不交给模型自由发挥，由人定结构骨架 + AI 在约束内填台词 + reviewer 按判据审。**

---

## 1. 对话长度 = 权力地位（最硬、可半机检的一条）

网文对白的**第一硬规则**：在一场权力不对等的对话里，**掌控者话短且冷，被压制方话多且情绪化**。短句方=权力上位，长句方=权力下位。这把"谁强谁弱"这种抽象关系，落成了一个**可统计的字数差**。

| 角色位置 | 台词长度 | 动作描写 | 为什么 |
|---|---|---|---|
| **掌控者 / 主角亮底牌** | **≤ 10 字** ⚠️经验值可调 | 不加 | 短=笃定=不屑解释；多一个字都是泄气 |
| **被压制方 / 急于辩解者** | **≥ 20 字** ⚠️经验值可调 | 可加（攥拳/咬唇/站起来） | 长=失控=需要靠音量找回场子 |

**典型对照**（压制模式）：

```
"你以为你是什么东西？这个家轮不到你说话！你嫁进来那天起就该明白自己的位置。"   ← 下位，长，情绪化
"滚。"                                                                      ← 上位，1 字，不加动作
```

**机检/reviewer 分工（关键，照 §0"能 code 判的绝不靠模型自觉"）**：

- **半机检（统计层，确定性）**：对白占比、单段最长 quote 字数、对话标签密度——都是纯计数（对照 `09` 半机检阈值清单，与"对话标签密度 ≤30%/30-50%/>50%"同层；属可补进 `output_check.py`/`antislop_lint.py` 的轻量统计，当前脚本已实现的是禁用词/弱化副词/段落方差，对白类统计是可补项）。
- **"谁是上位"= 章纲 + reviewer**：字数差只有在知道"谁该短"时才有意义。压制/反转 beat 里，**章纲可选标出本 beat 的 `power_holder`（谁掌控）**（见 `chapter-outline-template.json` 的可选字段）；标了之后"上位角色台词均长 ≤ 下位角色台词均长"是可比对的——但**当前脚本未实现对白均长统计，这条目前归 reviewer/人判**（power_holder 是给 reviewer 的锚，不是机检硬门）。脚本化是日后可补项，别当成已落地的机检。

> 对接 `07 §5` 打脸四部曲第 3 段"反转"：主角亮底牌时**把话说死、不绕潜台词**（这恰恰要求台词短、直、狠），与本节"上位≤10 字"是同一条规则的两个面。

## 2. 三种权力对话模式（压制 / 反转 / 心死）

§1 的长度差有三种最高频的结构化用法，writer 按章纲选一种填：

| 模式 | 结构 | 对应剧情 beat | 挂载契约 |
|---|---|---|---|
| **压制模式** | 对方长篇大论（3-5 行）→ 主角一字回应 | 碾压 / 打脸的"余震"段 | `07` 打脸四部曲·余震；情绪债释放 |
| **反转模式** | 对方嚣张（2-3 行）→ 主角一句话事实 → 对方沉默 | 亮底牌 / 真相揭穿 | `07` 打脸四部曲·反转（把话说死）；`cognition.misbeliefs`（对方的误判被一句话戳破） |
| **心死模式** | 对话越回越短：辩解 → 沉默 → "随意" | 关系破裂 / 虐心 | `emotion-debt` 虐点登记；情绪连续性（不能跳步） |

```
反转模式：
"你有什么资格管？这是我家的钱，我想怎么花怎么花。"
"你妈的存折，密码是我的生日。"      ← 一句事实，对方无话可接

心死模式：
"你听我解释，那天不是你想的那样。"   "嗯。"
"真的，我可以证明。"               "随意。"   ← 越回越短=情感账户清零
```

> ⚠️ 模式不能混：**愤怒式打脸**（你侮辱我→我打回去）和**瞧不起式打脸**（你看不起我→我证明给你看）情绪基调不同，混用会泄气（同 `07 §7`"压点写串"）。这条归 reviewer 看基调一致性。

## 3. 潜台词与议程（张力真正的来源）

AI 写对话最大的毛病不是"长度不对"，而是**所有角色立场一致、互相补全信息**——这等于复述，零张力。修法只有一条：**每个角色进场时带自己的议程，让议程碰撞。**

**两条规则：**

1. **议程（agenda）**：每个角色进入这场对话**想从中得到什么**。双方议程**一致**（同立场）= 复述，必须删或改成有人藏着别的目的；双方议程**碰撞**才是张力。
2. **潜台词（subtext）**：角色的**真实动机绝不浅显地写在台词里**。现实中人说话都给自己找借口——每句对白同时设计「动机（角色自己可能都没意识到）」+「借口（说出口的那层）」。

**议程从哪来（挂 typed 人物卡，不是凭空设）：**

| 议程/潜台词的来源 | 人物卡字段 | 怎么用 |
|---|---|---|
| 这场对话的目标 | `current_state.goal_now` | 角色为达成 goal_now 而说话，不是为了交代剧情 |
| 他知道 / 不知道什么 | `cognition.knows` / `does_not_know` | 信息差驱动他试探、隐瞒、误判 |
| 他**错信**什么 | `cognition.misbeliefs` | 反派据 misbelief 嚣张（"以为主角是废柴"），台词的底气来自错误前提 |
| 读者知道但他不知道 | `cognition.reader_knows_char_doesnt` | **戏剧反讽对话**：他说的每句话读者都听出双关 |

**语气 = 关系 × 场合 × 目的**：私密的话放到**公众场合**说才有冲击力（当众翻脸）；对话里不能完全说出口的，用动作/神态/环境补。

> 全归 **reviewer 语义判**（议程是否碰撞、动机是否藏在潜台词里、有没有戏剧反讽）——这层无法机检，是 `dialogue_natural` 的核心扣分点之一。

## 4. 7 维人物语言差异化（+「遮名识人」自检）

"所有角色说话一个味"= 人设模糊的直接症状，也是 AI 味重灾区。每个主要角色要在 **7 个维度**上有可辨识的语言指纹：

| 维度 | 操作 | 挂哪个人物卡字段 |
|---|---|---|
| 口癖 / 惯用语 | 给每个主要角色一个标志性用词 | `voice_notes`（口头禅） |
| 说话节奏 | 长篇大论 vs 短句连击 | `voice_notes`（语速） |
| 信息偏好 | 技术型带术语，江湖人带切口 | `behavior_anchors` / `identity` |
| 立场固定 | 永远从某角度发言（悲观/乐观/务实派） | `behavior_anchors` |
| 身份措辞 | 老者/少年/贵族/市井：自称、敬谦词不同 | `voice_notes`（第一人称、怎么称呼别人） |
| 性格语气 | 智谋型话里有话；鲁莽型想到就说；冷静型偶尔情感外露反成冲击 | `flaw_personality` / `behavior_anchors` |
| 关系进度 | 初见/熟悉/对立/亲密，语气与信息量不同 | `current_state.relations[].stage` |

**长篇连载专属适配（短篇没有的问题）**：单篇里角色声音只需在几千字内自洽；**连载里角色要在几十、上百章里保持同一把嗓子**。所以 `voice_notes` 是人物卡的 **Canon 慢变字段**，必须**每卷/风格锚定时重注入 writer**（同 `03` 风格锚每章重注入），否则角色语气会在长跨度里漂移成"全书一个味"——这是长篇特有的退化，归 §0 的"工程能修"那一类。

**唯一硬自检（reviewer 必跑）**：

> **遮住角色名，能否只凭台词区分是谁在说话？** 不能 → `dialogue_natural` 直接扣到最低线附近。这是 7 维差异化是否落地的总验收，纯语义、归 reviewer。

## 5. 弹幕 / 群众对话——三层递进

群众弹幕（围观、评论、震惊）是网文放大爽点/燃点的标配。**没设计的弹幕**只有满屏"卧槽好厉害"，单调浪费；**设计过的弹幕**是一条**递进链**：

```
普通人震惊  →  专业人士分析  →  特殊身份者反应  →（情感升华）
"这怎么可能"    "他用的是失传的…"   "连他都惊动了？"      "原来这就是…"
```

**三件核心作用**：① 剧情推进（透露主角不知道的信息）；② 增加期待感（悬念、猜测）；③ 情绪渲染（群众震惊传染读者）。

**操作要点 / 适配：**

- 每条**短小精悍**、不同人格化语气，不能每条一个味（这是 §4 差异化在群众层的延伸）。
- 可埋一个**"反转"角色**：看似路人一句话改变所有人认知。
- **不每章都写**——集中在关键爽点/燃点/泪点前后 → 直接挂 `sao_schedule`：弹幕密度跟着爽点排布走，不是均匀撒。
- ⚠️ 弹幕是**锦上添花，不代替主线**；用它注水=反模式。

> 归 **reviewer 语义判**层次感（普通→专业→特殊身份是否成立、有没有降格成满屏复读）。

## 6. 角色不当"科普嘴"（并入 info-dump 裁决）

**铁律**：设定 / 原理 / 前因后果**不能靠任何角色（尤其信息型配角、AI 伴侣、解说役）整段讲解**。让某个角色站定背一段世界观，和旁白大段 info-dump 是**同一种病**，只是换了引号——**`09` 的 Gate G（解释腔·上帝视角·安排感）同样适用于角色台词。**

**正确写法**：把设定拆成**角色在压力下挤出的半句话 + 身体反应 + 留白**，用到哪点带哪点，不需要把前因后果讲全（同 `03` 防泄漏四件套：设定块声明"仅为参考，严禁在正文复述"——角色台词复述设定 = 变相泄漏）。

**反例 → 修法：**

```
✗ 科普嘴："你要知道，这片大陆分为九大宗门，三千年前因为灵脉枯竭引发了…（整段）"
✓ 压力半句："九大宗门……" 他喉咙一紧，没说完，"——别提那三个字。"  ← 设定带出来一角，留白
```

> 归属：**reviewer 语义**（台词是否在替作者讲设定）+ `output_check.py` 解释腔启发式（词面）。与 `world_canon` info-dump 裁决合并，不单独立项。

## 7. 反模式（对话写崩的几种死法）

| 反模式 | 症状 | 根因 | 归属 |
|---|---|---|---|
| **审讯式** | 通篇一问一答 | 没人主动说，全靠提问驱动 | 改"一方主动说+另一方给动作/表情反应"；reviewer |
| **配角无脑夸主角** | 假、油 | 用对话喂爽点免费（同 `07` 爽点免费） | reviewer |
| **互相解释读者已知信息** | 水字数 | 双方议程一致=复述（§3） | 删，或改叙事一句话概括；reviewer |
| **大段说明文式对话** | 闷 | 科普嘴（§6） | Gate G；reviewer + output_check |
| **所有角色一个味** | 人设模糊 | 7 维差异化没落地（§4） | 遮名识人自检；reviewer |
| **对话说服人物** | 假——现实中没人被几句话说服 | 用台词强推剧情 | 改用突发状况代替说服；reviewer |
| **刻意每句配动作/表情** | 行文机械 | 把"去 AI 味"误解成多加描写 | 动作只在关键转折配；对接 `09` Gate E |

> 对接 `09` **Gate E（对话去腔调）**：每句信息完整、逻辑太顺、对话标签密度过高、书面腔——加口语（"嗯/行吧"）、答非所问、动作穿插、删解释性对话。本节与 Gate E 方向一致，互为词句层（E）与结构层（本节）。

## 8. 与生产循环的接口（挂载点）

| 步骤 | 谁 | 用本文什么 | 落到哪 |
|---|---|---|---|
| **步骤4 · writer（生成正文）** | automate（关 thinking，吃章纲+人物卡切片） | §1 长度差、§2 模式选型、§3 议程（从 `goal_now`/`cognition` 取）、§4 `voice_notes` 重注入、§5 弹幕跟 `sao_schedule`、§6 不背设定 | 章节正文草稿 |
| **步骤6 · reviewer（审 `dialogue_natural`）** | automate（独立上下文/异模型） | 把 `dialogue_natural`(max15/min9) 的判据展开为 4 条子检：**对话长度=权力**（上位是否更短）/ **弹幕递进**（普通→专业→特殊身份）/ **遮名识人**（遮名能否区分）/ **占比≤40%** ⚠️经验值可调 | `review-report` `weighted_scores.dialogue_natural`；跌破 min=9 → REVISE |
| **机检层（半机检，确定性）** | code | 对白占比%、最长 quote 字数、对话标签密度、（章纲标 power_holder 后）上位 vs 下位台词均长比对 | 挂 `output_check.py`/`antislop_lint.py` 半机检统计层 |

**可机检 vs 归 reviewer（边界，照 §0）：**

- **可机检（统计，code）**：对白**占比 ≤40%** ⚠️经验值可调、单段最长台词字数、对话标签密度、上位/下位字数差（**前提：章纲标了 power_holder**）。
- **归 reviewer（语义，无法 code）**：**谁是权力上位**、**议程是否碰撞 / 潜台词是否藏住动机**、**遮名识人**、**弹幕三层递进是否成立**、**有没有科普嘴**、**情绪基调是否写串**。

> 一句话边界：**"字数、占比、标签密度"是机检的；"谁该短、议程碰没碰、是不是一个味、台词在不在替作者讲设定"是 reviewer 的。** 机检只负责把统计异常标出来，质量裁决始终在 `dialogue_natural` 加权项里。
