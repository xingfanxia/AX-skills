# 05 · 品类约束模板库

> Purpose: 把"网文结构强品类依赖"落成**第 0 步锁品类后可一键挂载的约束模板/检查清单**——每个主流品类给六件套（①爽点引擎单句锚 ②开篇黄金三章必交代项 ③结构骨架类型 ④数值/state schema 要点 ⑤毒点黑名单 ⑥AI 最易写崩点 + 工程兜底）。**用在阶段0（SKILL.md §2.2 锁定配置层）**：人类拍板 `genre/subgenre/tone` 后，把对应品类块的六件套填进 `templates/contract-template.yaml`（`config.*` + `contract.sao_engine` + `power_floor_ceiling` + `forbidden` + `locked_reveals`）与 `templates/state-world.yaml`（`power_system` / `glossary`），并把毒点黑名单接到 reviewer 硬门 `no_poison_points`（见 `references/04-review-rubric.md` §1）。平台维度的开篇阈值/毒点容忍另见 `references/06-platform-params.md`。

---

## 0. 原理：品类即配置，六件套各自落到哪个文件

网文不是"通用写作 + 换题材皮"，而是**每个品类自带一套硬一致性约束**（玄幻要境界表、悬疑要真相金库、种田要经济账本）。AI 在每个品类的崩点**高度可预测且大多是硬一致性问题**（战力通胀/真相泄漏/经济崩），所以可用显式 schema + 可见性分级 + 毒点 lint **自动兜底**；而走向/节奏/爽点拍板是 AI 最弱项，须人类主导[chinawriter/40370888; cloud.tencent/2650227]。本文把"品类知识"沉淀成可挂载配置，避免每开新书重新口头约定。

| 六件套 | 落到哪 | 谁消费 |
|---|---|---|
| ①爽点引擎锚（单句） | `contract.sao_engine` | 注入每章 prompt（compiler）；reviewer 验 `sao_payoff` |
| ②开篇黄金三章必交代项 | `config.opening_rules` + 第一卷 arc beats | 开篇 outliner；`output_check.py` 的 `opening_ok` 门扫"第N字前出冲突/金手指"（正文输出侧·启发式） |
| ③骨架类型 | `config.skeleton_type` | planner 选大纲生成器（§1） |
| ④数值/state schema | `state-world.yaml.power_system` / 经济账本 / 真相金库 | `state_check.py` 校验单调性/缺口；continuity-checker |
| ⑤毒点黑名单 | `contract.forbidden` + 词面黑名单 | reviewer 硬门 `no_poison_points`；按平台切容忍度（→06） |
| ⑥AI 最易崩点 + 兜底 | 决定该品类强化哪条校验 | driver 配置该品类的重点 gate |

> 铁律重申：毒点/战力上限/真相可见性这些 **MUST-hold 不变量必须由 schema + code 强制**（`state_check.py` / 可见性截断 / 黑名单扫描），**绝不能写进 prompt 让 LLM 自觉**——那是会随模型漂移的伪约束（SKILL.md §0 铁律4）。

## 1. 先选骨架（`skeleton_type`）：三类，决定用哪个大纲生成器

骨架类型比品类更底层——**同一骨架的大纲生成逻辑相同**，品类只是往骨架里填爽点引擎和数值。planner 先据此选生成器[chinawriter/40370888 数据库化; vocus/685aaa 全职高手三段链]。

| `skeleton_type` | 覆盖品类 | 大纲生成器逻辑 | 一致性负担 | 对应字段 |
|---|---|---|---|---|
| **升级阶梯** | 玄幻/仙侠/修真升级流·系统流·末世进化·电竞段位 | 全书拉 3 段大情节（第一段≤30万字解决）+ 换地图节点；线性单调推进 | **高**（跨章战力/境界单调性极易通胀）→ 重 `power_system` 校验 | `power_system.ladder` + `monotonic:true` |
| **单元剧+长线暗线** | 无限流/规则怪谈·都市打脸·悬疑案件·诸天经营·网游副本 | 单元（副本/案/打脸周期）为骨 + 一条长线暗线串联；每单元三段式（预热→爆发→回味） | **中**（单元内自洽 + 暗线不能割裂）→ 每单元独立校验 + 暗线单独记忆线 | `mainline_beats` 主线 + `sidelines` 暗线分管 |
| **慢热长线经营** | 历史/种田/经营·领主流·部分大女主事业线 | 资源积累曲线为骨，低开高走；允许慢热但主线不停滞 | **中**（数值前后矛盾 + 慢热写成注水）→ 经济账本 + 每章获得感锚点 | 经济账本（见 §7）+ `reader_promise` 每章1获得感 |

> ⚠️ 经验值：单元周期长度（打脸/副本每几章一轮）随平台显著不同，见各品类块与 `06-platform-params.md`，作可调参数不硬编[wangwen666/186]。

## 2. 玄幻 / 仙侠 / 修真（升级流）

> **爽点引擎锚**（填 `contract.sao_engine`）：「由弱到强逐级升级，抑后扬，**升级后装逼打脸 + 降维碾压**」——升级本身是小爽点不要太多笔墨，读者真正期待的是**升级后的打脸**，打脸要拉长高潮用围观者衬托[ady101 林海听涛; write.qq/5337117]。
> **子类**（填 `config.subgenre`）：凡人流 / 洪荒 / 签到流 / 苟道 / 强者重生 / 夺舍废材 —— 起点把主角分废材崛起/土著成长/强者重生/夺舍废材四型，均"由弱到强、抑后扬"[write.qq/5337117]。

| 维度 | 内容 |
|---|---|
| **开篇黄金三章必交代** | 主角困境（被打压/废材身份）+ 金手指（功法/外挂，起点可第三章前亮、番茄前1000字亮）+ 力量体系第一级 + 一个小升级或小打脸兑现卖点 |
| **骨架 `skeleton_type`** | 升级阶梯：3 段大情节（30万字前收第一段）+ 换地图（宗门→秘境→更大世界）|
| **数值/state schema** | **必填 `power_system`**：`ladder`（练气→筑基→金丹…，顺序即单调约束）+ `cross_tier_rule`（越级边界）+ `cost_rule`（升级代价，无代价=毒点）+ `ceiling`（全书最高战力，**开局锁死**写进 `contract.power_floor_ceiling.ceiling`）。`glossary` 钉死功法/境界专名唯一写法。`state_check.py` 每章校验境界单调不倒退 |
| **毒点黑名单**（→`contract.forbidden`） | 战力崩坏/越级无代价；**吃书**（配角上一章筑基下一章练气）；境界倒退；到宗门后"套娃循环、水漫金山"注水；无代价升级；圣母/油腻种马（白文通杀毒点）|
| **AI 最易崩点 + 兜底** | **战力/境界体系通胀 + 吃书**（最高频）。兜底：`power_system` 阶位表 + `monotonic:true` 让 `state_check.py` 在每章生成后机械校验单调性，**不靠模型记境界**；大纲按"三段大情节 + 换地图"节点产出，禁线性堆章[cloud.tencent/2626811] |

> 仙侠/古典仙侠**调性特例**：语言要"有古气、偏文青"，**辞藻在此是卖点不是毒点**——`config.tone` 设"古气文青"后，去 AI 味词句层（桶1）对该书放行辞藻（见 `09-anti-ai-slop.md` 桶1 品类适配，推翻文学清单第4条）[aipiaxi; qidianclub/1882《剑来》争议]。

## 3. 都市（赘婿 / 战神 / 神医 / 兵王 / 系统流）

> **爽点引擎锚**：「**隐藏身份揭露 + 打脸四部曲（挑衅→隐忍→反转→余震）+ 扮猪吃老虎**」——爽点集中在身份揭露、系统、扮猪吃虎，爽点越密越易追读[zhihu/3573227; chinawriter/40293380]。系统流子类锚另写：「**金手指显形化 + 不讲道理的升级感 + 规则之外但有稳定预期的规则**」[qidianclub/125; chinawriter/40370888]。

| 维度 | 内容 |
|---|---|
| **开篇黄金三章必交代** | 主角隐藏身份的"猪"皮（落魄赘婿/普通人）+ 一次小打脸兑现卖点 + 真实实力的暗示但不揭穿。飞卢前300字必有炸点、3章一周期；番茄前500字出冲突 |
| **骨架 `skeleton_type`** | 单元剧+长线暗线：打脸单元循环为骨，一条"身份终极揭露"长线暗线。**周期章数=平台参数**：番茄5-8章/起点15-20章/飞卢3章一周期 ⚠️经验值[wangwen666/186] |
| **数值/state schema** | **隐藏身份/真实实力登记为对 writer 隐藏的可见性条目**：写进 `contract.locked_reveals`（reveal_at_volume）+ 相关 canon 设 `visible_from_volume > 当前卷`，每章章纲 `forbidden_reveals` 列本章禁泄项。打脸单元用"挑衅-隐忍-反转-余震"四段卡（对照 `sao_payoff.setup_chain`）。系统流另需**系统规则契约文件**（奖励公式/触发条件/限制），写进 `power_system`，禁 AI 临场改规则 |
| **毒点黑名单** | **马甲过早泄漏**（毁反转，本质=剧透泄漏）；打脸节奏写得太均匀（每次等强度）；反派降智；系统大量无用对话拖慢节奏（注水）；绿帽/送女（飞卢零容忍）|
| **AI 最易崩点 + 兜底** | **隐藏身份过早泄漏**（毁掉整本反转，最高频）+ 打脸均匀化。兜底：可见性分级在 compiler 端**截断**真实身份进 prompt（不靠模型"记得保密"）；reviewer 硬门 `no_forbidden_reveal_leak` 复检；节奏在章纲显式标注本周期"挑衅/隐忍/反转/余震"哪一段，AI 不得自定强度[wangwen666/186; chinawriter/40293380] |

## 4. 无限流 / 规则怪谈

> **爽点引擎锚**：「**主角解读并利用规则破局 + 恐怖氛围 + 解谜反转**」——规则怪谈的核心魅力恰恰是**规则的不可靠**，主角在矛盾规则中求生[lkong; write-on-paper/lun; bjnews《十日终焉》数据库化]。

| 维度 | 内容 |
|---|---|
| **开篇黄金三章必交代** | 主角被卷入副本/规则体系的机制（怎么进、死了怎样、奖励是什么）+ 第一个副本的规则清单 + 一次"读规则漏洞破局"小爽兑现卖点 |
| **骨架 `skeleton_type`** | 单元剧+长线暗线：副本制，每副本三段式（起因/经过/结果 = 爽点预热/爆发/回味）+ 一条"幕后体系真相"长线暗线 |
| **数值/state schema** | **每个副本生成独立"规则清单 + 漏洞（破局点）"结构化卡**，作为该副本的局部 canon，**对该副本内一致性单独校验**（不串别的副本）；可做成"事件卡数据库"供人工挑选。**主线暗线与副本明线分两条记忆线管理**（暗线进 `sidelines`，副本卡进局部 state）|
| **毒点黑名单** | 副本规则内部不自洽（同一规则前后矛盾，非"故意矛盾"那种设计）；副本与主线割裂（每副本=孤立小说）；恐怖氛围被爽点稀释（破局太轻松不吓人）；规则破局靠主角天降灵感而非读者可推 |
| **AI 最易崩点 + 兜底** | **单副本内规则自洽**（AI 易在副本中途忘记自己定的规则）。兜底：副本规则卡作为局部 canon 注入该副本所有章 + 副本结束做一次该副本闭环校验；单元化天然把跨章一致性负担压到副本长度内，是 AI 最能稳的结构[lkong"每副本结束掉追读要提前留钩子"] |

## 5. 科幻 / 末世（丧尸 / 废土 / 星际）

> **爽点引擎锚**：「**艰难求生 + 基建/丛林法则 + 进化升级**」——`tone` 决定底色：「艰难求生」是末世正路（"主角金手指大开大杀四方就不叫末世"），「爽囤货」是另一调性，由人类选定后影响全书资源稀缺度参数[sohu/512862136; cl.ntpu 末世论文]。

| 维度 | 内容 |
|---|---|
| **开篇黄金三章必交代** | 末世降临的机制 + 主角的生存优势/进化金手指 + 第一次危机求生 + 资源稀缺基调（艰难求生调性）|
| **骨架 `skeleton_type`** | 升级阶梯（进化体系即升级阶梯）：进化等级换地图（安全区→丧尸潮→更大灾变）|
| **数值/state schema** | **进化体系同升级流**：`power_system.ladder` + `ceiling`（最高进化等级/战力上限**开局锁定常量**，`state_check.py` 校验单调）。资源稀缺度作为 `tone` 派生参数（产出 vs 消耗的缺口在"艰难求生"调性下恒为负）|
| **毒点黑名单**（末世专属，可直接做 lint） | **父母双全**（限制主角行动）；**后宫开太多**（剧情乱套）；**敌人不死光环**；**政权/政府无理由无差别崩溃**；进化体系不锁最高战力（同升级流通胀毒点）[sohu/512862136 逐条] |
| **AI 最易崩点 + 兜底** | 进化战力通胀（同升级流）+ 把"艰难求生"写成主角无双爽囤货（背离调性）。兜底：`ceiling` 锁死 + 单调校验；`tone=艰难求生` 时把"父母/后宫/敌人复活/政府状态"四条写进词面黑名单做自动 lint，命中即 reviewer 硬门 `no_poison_points` false |

## 6. 悬疑 / 推理（含叙述性诡计）

> **爽点引擎锚**：「**揭秘/反转的那一刻 + 叙述性诡计的信息差**」——叙诡=作者利用结构/文字技巧对读者**刻意隐瞒/误导**关键信息，真相揭晓前作者完全站在凶手一方[baike 叙诡; bjnews/164757267]。

| 维度 | 内容 |
|---|---|
| **开篇黄金三章必交代** | 一桩勾人的案/谜（钩子）+ 侦探/主角的能力 + 公平性承诺（读者隐约感到"线索都在"）。**注意：叙诡品类开篇要藏的信息最多，开篇阈值与爽文不同**（→06 起点慢热侧）|
| **骨架 `skeleton_type`** | 单元剧+长线暗线：单元案为骨 + 一条长线主谜（贯穿全书的大诡计/真凶）|
| **数值/state schema** | **本品类不可省的子系统=真相金库 + 线索台账**（最强可见性分级）：诡计核心、凶手身份、关键隐瞒项标记为**对 writer 隐藏**（`locked_reveals` + `visible_from_volume` 极高 + `cognition.reader_knows_char_doesnt`），仅向大纲层开放；公平性线索按计划埋入用 `foreshadow-ledger` 台账确保"埋了 + 按卷回收"[baike; 新京报"精准控制何时揭露"] |
| **毒点黑名单** | 提前泄底（"点一句话就泄底"）；公平线索没埋（读者骂"猜不到也没法猜"）；真凶降智/侦探开挂直接知道答案；诡计逻辑漏洞 |
| **AI 最易崩点 + 兜底** | **AI 不知道哪些信息该藏，极易提前泄底或埋错公平线索**（本品类头号崩点，因为叙诡要全局性刻意隐瞒，与 AI"想到啥写啥"天然冲突）。兜底：真相金库的可见性分级在 compiler 端**硬截断**——writer 拿不到真相，自然写不漏；线索埋设由人/半自动经 `foreshadow-ledger` 登记，writer 只按章纲 `due_foreshadows` 埋指定线索，不自由发挥[baike 叙诡；这是该品类必上可见性分级的根本原因] |

## 7. 历史 / 种田 / 经营（领主流 / 诸天经营）

> **爽点引擎锚**：「**获得感 + 资源积累 + 缓慢成长的满足**」——爽点是"获得感"，硬规则是经济体系不能崩[bilibili/青狐 种田总结]。

| 维度 | 内容 |
|---|---|
| **开篇黄金三章必交代** | 主角的处境与经营起点（一块地/一个领地/一项金手指）+ 金手指的**限制**（不能开太大）+ 第一笔获得感（低开）。低开高走是种田正路[qidianclub/694《黎明之剑》] |
| **骨架 `skeleton_type`** | 慢热长线经营：资源积累曲线为骨，低开高走；诸天经营叠"升级换地图"则混入单元剧 |
| **数值/state schema** | **必填经济账本**（硬规则）：「**收获永远跟不上消耗**」（产出 < 消耗的缺口恒在）+ 金手指限制清单。`state_check.py` 校验数值单调性与缺口恒在。**允许慢热但设 `reader_promise`：每章至少 1 个获得感锚点**防注水[bilibili/青狐"经济体系不要崩、金手指要有限制否则期待感没了"] |
| **毒点黑名单** | **慢热写成注水流水账**（主线停滞）；资源/数值前后矛盾；金手指开太大（期待感崩）；后期"升级换地图重复式装与打脸、无力掌控烂尾"（诸天经营通病）[163/HRH72UPV] |
| **AI 最易崩点 + 兜底** | **注水流水账 + 资源数值前后矛盾**。兜底：经济账本做缺口恒在校验（产出始终 < 消耗，`state_check` 扫）；`reader_promise` 每章 1 获得感锚点 + reviewer `pacing_density` 验主线推进，防慢热=停滞 |

> ⚠️ **跨文件缺口**：`templates/` 当前只有 `power_system`（升级阶梯用），**无专用经济账本 schema 文件**。种田/经营开书时需在 `state-world.yaml` 自定义 `economy` 块（产出/消耗/缺口/金手指限制清单）并让 `state_check.py` 扩展"缺口恒在"校验——这是该品类落地前要补的一步。

## 8. 女频（言情 / 古言 / 现言 / 穿越 / 宅斗）

> **爽点引擎锚**：「**情感拉扯/修罗场 + 重生复仇打脸虐渣 + 大女主事业线**」——女频正进入"言情+X"与"去女频化"时代，强化事业线，"言情+权谋/悬疑/仙侠"成主流[chinawriter/40034334 去女频化; 肖映萱反言情升级脉络]。

| 维度 | 内容 |
|---|---|
| **开篇黄金三章必交代** | 女主处境与渴望 + 情感关系的初始张力或重生/穿越的信息差优势 + 一次小复仇/小打脸或一次情感推进兑现卖点 |
| **骨架 `skeleton_type`** | 双线并行（言情主线 + X 副线）：结构沿"宅斗→宫斗→权谋"升级；大女主事业线偏慢热长线经营 |
| **数值/state schema** | **言情主线 + X 副线（权谋/事业/悬疑）双线节奏约束**：`mainline_beats` 拆两条，**每 ≤5 章一次情感推进**（情感线是最软的爽点，写进 `reader_promise`）。**重生者/穿越者"已知未来信息"登记为可见性条目防穿帮**（`cognition.knows` + `visible_from_volume`）。宅斗群像用人物关系图谱专项管理（多人物关系网）|
| **毒点黑名单** | 圣母/白莲花女主；感情线推拉无铺垫直接 CP（晋江三观+感情线审核最严）；重生/穿越信息差前后矛盾；配角降智；男主前期戏份处理不当（古言常见结构吐槽）[hongxiu/13304] |
| **AI 最易崩点 + 兜底** | **①感情线推拉节奏**（最软的爽点，human-only 高软度部分，AI 最难拿捏推拉火候）；**②重生/穿越者信息差一致性**（易让女主"知道她还不该知道的事"）；**③宅斗多人物群像关系网**易乱。兜底：情感推进由人在章纲标注力度，AI 填空；信息差用可见性分级 + `cognition` 边界硬控；群像用关系图谱做 continuity 校验[chinawriter/40034334] |

## 11. 开篇建构手册（借鉴 oh-story-claudecode）

> 取材：oh-story-claudecode `opening-design.md`（MIT）。**定位**：§2-8 各品类块已给"开篇黄金三章**必交代什么**"；本节补"**怎么建构**那三章"的工艺，全部落到 `config.opening_rules`（开篇契约）+ **第一卷 arc beats**（outliner），不新增硬门。开篇阈值的**平台维度**（前N字出冲突/亮金手指）唯一事实源仍是 `06-platform-params.md`，本节不复制。

### 11.1 三大基点 + 困境/优势两种切入

开篇前三章必须埋三个"基点"，缺一则代入感/期待感不足——把它们作为第一卷 arc beats 的**必达项**写进 `config.opening_rules.must_seed`：

| 基点 | 干什么 | 完成时机 ⚠️可调 |
|---|---|---|
| **人设基点** | 展示主角核心性格+处境 → 建代入/共情 | 前 3 章 |
| **切入点基点** | 主角遭遇的第一个冲突/机遇 → 制造紧迫感、必须**跟主线强相关** | 最好第 1 章 |
| **金手指基点** | 展示主角独特优势 → 制造期待+差异化 | 前 3 章 |

**切入顺序由调性决定**（写进 `config.opening_mode`）：

- **困境切入型**（先人设）：①不利处境（人设=共情入口）→②压迫/危机（切入点=紧迫感）→③金手指激活（反转期待）→④初步解决但引出更大问题（章末 `ending_hook`）。
- **优势切入型**（先金手指）：①已有优势但被低估/隐藏（人设+金手指合一）→②优势意外暴露（切入点）→③周围认知刷新（震惊）→④暴露引来更大麻烦（`ending_hook`）。

> 对齐：①②③④的"引出更大问题/麻烦"就是第一章章末钩子，型号取 `ending_hook.type` 四型之一（§07）；"周围认知刷新"是都市/扮猪吃虎的 `sao_payoff`（§3 打脸四部曲的"反转"段）。

### 11.2 噱头三分法（吸量后必嫁接主线）

噱头负责"点击+追读承诺"，目的达到后**必须立刻嫁接主线**，否则噱头一过即掉读：

| 噱头类型 | 写法要点 | 落到契约 |
|---|---|---|
| **事件噱头** | 书名即事件，约 5 章；一上来就进事件，**不铺垫穿越/金手指** | `config.opening_mode: event`；第一卷 beats 前 5 章为"事件弧"，第 6 章起接主线 |
| **金手指噱头** | 先写主角+困境，再引出金手指；分布全文前多后少 | `config.opening_mode: cheat`；与 `power_system` 首级绑定 |
| **人设噱头**（高复杂度） | 仅当该人设能**持续驱动**剧情冲突时用；单纯"杀伐果断"不算出彩人设 | 慎选；需 `behavior_anchors` 能持续产出戏剧性才成立 |

> **开书前评估**：噱头能否延伸出后续更多字数？不能延伸 → 用"噱头引流 + 常规题材"策略（开头噱头只吸量，正文走常规）。这条决定第一卷之后 `skeleton_type` 怎么接（§1）。事件写法与金手指写法**不能混用**。

### 11.3 书名 / 简介 / 开篇三位一体（一致性裁决）

- **书名**告诉读者金手指 + 题材；**简介**只写三件事（主角处境 / 金手指能干嘛 / 第一个爽点预览）；**开篇**必须呼应书名+简介的承诺。
- **一致性违规**：书名写"苟"但主角张扬、书名/简介承诺 A 但开篇给 B —— 这是**顶层卖点漂移**，挂 reviewer 自检项（非硬门，语义判定）+ 写进 `contract` 顶层"卖点一句话"供每卷比对。
- **书名翻车预警**：三字四字看不出亮点是大忌；不要拿悬念当期待感——直接告诉读者"能干嘛"。
- **卖点显隐分层**：商业性卖点（噱头/新鲜金手指）vs 文学性卖点（文风/世界观）；平台"海水"越大越需多卖点叠加（番茄 > 起点 ⚠️可调，→§06）。**五步验证**：①核心梗不能以"升级/爽文"为卖点 →②围绕核心特化世界观 →③人设反差 →④个人写法特色 →⑤蹭热点。

### 11.4 开篇章内字数节拍 + 情绪波动线（⚠️均为经验值，可调）

> 这是**开篇章节内部**的节拍模板（约 2500 字铺第一个完整爽点），不是"全书字数 %"——长篇请只把它用在第一卷的开篇章，后续按卷/arc 节奏推进。

| 段落 | 字数 ⚠️可调 | 内容 |
|---|---|---|
| 世界观+穿越事实+前身经历 | ~300 | 快速交代背景，禁大段设定解说 |
| 第一个吸睛小剧情 | ~500 | 用事件制造初始吸引 |
| 前置金手指获得安全感 | ~400 | 金手指首次展示 |
| 感受到不安全 | ~400 | 新威胁出现 |
| 遇见危机 | ~500 | 核心冲突到来 |
| 直接使用金手指 | ~400 | 解决问题，完成第一个爽点 |

**情绪波动上行/下行交替法**（每次波动都要有**具体信息支撑**，共鸣是关键步骤）：穿越成仙人（上行）→只是记名弟子（下行）→杂活背锅（触底+共鸣）→有金手指了（上行）→金手指是同归于尽的（下行+戏剧性）。这条与 `reader_emotion_curve`（章纲）同源——把开篇的上下行写进第一卷 beats 的情绪曲线。

**开头禁忌**（命中即 `opening_ok` 启发式 warn，正文输出侧）：大段背景介绍 / 天气风景开头（除非反差极大）/ 一次出场 3+ 主要角色 / 序章楔子（除非极简且与第一章逻辑紧密）/ 插叙切视角回忆梦境 / 世界观详细解说（至少等第一个一级结构结束）。

## 12. 剧情结构骨架与循环（plot-structure 折入 · 借鉴 oh-story-claudecode）

> 取材：oh-story-claudecode `plot-core-methods.md` + `genre-catalog.md`（MIT）。**定位**：§1 已定"选哪个 `skeleton_type`"；本节补"骨架内部**怎么搭剧情、怎么循环、怎么不崩**"的工艺，挂在 `mainline_beats`/`sidelines` + `emotion-debt`(`sao_schedule`) + `power_system` 之上，**不另起 plot ref**。

### 12.1 主线的正确定义（顶层锚点别写错）

**主线 ≠ 升级**。主线是**一件事**（斗破=上云岚宗复仇），升级只是达成它的行动。挂 `contract` 顶层"主线锚点"——锚点写成"升级"或元素罗列，后续所有剧情会偏差。主线完成后：要么铺垫开启第二条主线，要么完结。

### 12.2 双线明暗 + 场景切换钩子

明线（当前单元/副本）+ 暗线（幕后真相/长线）分两条管：明线进 `mainline_beats`，暗线进 `sidelines`（与 §4 无限流、§6 悬疑的"暗线单独记忆线"同构）。切换规则：**主线占比最大**；在悬念点切出（不在平淡处）；每条线切换前留一个钩子；过渡场景无信息量就跳过。

### 12.3 AB 粗纲交织 + 高潮逆推（beat sheet 方法，按卷/arc 用）

**这是吸收 `genre-writing-formulas` 的 beat sheet 方法本身，不搬它的 21 篇短篇成品**（那会把本库撑成短篇手册、偏离长篇定位）。长篇用法：**一卷 = 一个"中套娃"**，对每卷套一张 beat 表，而非按全书字数 % 排：

1. **高潮逆推**：先定本卷高潮爽点 → 反推所需人物/铺垫。
2. **AB 交替**：`A` = 压情绪/铺垫/埋伏笔；`B` = 抬情绪/小收获/小反转。ABABAB 排列，写作时只盯当前 AB 段。
3. **对齐情绪债**：`A` 段 = 开一笔 `emotion_debt`（积 `intensity`）；`B` 段 = 小额 `release`；本卷高潮 = 大额 `release`，受 `release_intensity >= intensity` 约束（§07 守恒律）。短篇公式里"开头前10%/发展10-50%/高潮50-80%"的全书 % → **一律改读成"卷内/arc 内占比"**，长篇按卷重置节拍。
4. **高潮五步公式**：蓄能 → **假胜**（先给希望再击碎，制造情绪落差）→ 崩解（伏笔一次引爆 + 推入单人绝境）→ 交叉死磕（对抗线/绝境线来回切）→ 悬置收尾（胜负不立刻揭晓）。**没有假胜的高潮缺情绪落差**。

### 12.4 六种经典循环模式 + 三要素统一

**循环公式**：`题材 + 金手指 + 主角身份 = 循环模式`，三要素必须统一（与 §0"循环从设定中诞生"同义）。选骨架（§1）时据此挑循环：

| 循环模式 | 机制 | 燃料 | 常见骨架 |
|---|---|---|---|
| 案件串循环 | 案→解谜→部分真相→更大谜团→新案 | 信息差+推理 | 单元剧+长线暗线 |
| 扮猪吃虎循环 | 发育→挑衅→碾压→震惊→继续发育 | 读者-角色信息差 | 单元剧+长线暗线 |
| 资源积累循环 | 资源→技能→实力→新地图→新资源 | 螺旋上升 | 升级阶梯 / 慢热经营 |
| 戏剧性反转循环 | 亏→反转赚更多→再投入→又赚 | **不依赖数值膨胀** | 慢热经营 |
| 组织枢纽循环 | 各自冒险→信息汇聚→衍生新剧情 | 信息交换+多线 | 单元剧+长线暗线 |
| 公路片循环 | 走一段→遇一人→又走→又遇 | 人物塑造力 | 单元剧 |

> **变体**：天才流→大幅增加等级数量防数值几十章就崩；无敌流→循环核心转为信息差（来一个秒一个，每次刷新认知）。换地图时循环要**升级**（更大规模/更高门槛/更强对手），并保留至少一条贯穿主线（复仇线/旧关系/信息差/提前铺垫）。

### 12.5 信息团（水章自检）

信息团 = 一段剧情要表达的核心信息单元，**每个必须能一句话概括"这段在讲什么"**。节奏快=信息团密集，水章=信息团为零或与主线无关。例：一章里"主角识破骗局"是 1 个信息团，"顺带交代反派背景"是第 2 个——两个无关就是水章，砍掉第二个或让它服务第一个。挂 reviewer `pacing_density` + 章纲 `chapter_purpose`（一句话写不出本章信息团 = 该章存疑）。

### 12.6 自嗨判定法（写给谁看）

写大剧情/卷纲后做三问，**有一问答不好 = 自嗨，立刻停下修正**：①这书写给谁看（目标读者年龄/职业/性别/平台/处境/渴望）？②他们看网文想看什么？③本书哪些内容是他们想看的？挂 `reader_promise`——自嗨判定是 `reader_promise` 的上游校验（连读者画像都模糊，`reader_promise` 必然空泛）。纠正：分析同类书高频正面评论关键词反向校验情节选择。

### 12.7 金手指变体：四阶段演进 + 升级差异化 + 骨相三型

**四阶段演进**（呈现力度随阶段递增，重心可转移但必须有足够伏笔支撑、可淡化不能完全抛弃）：

| 阶段 | 操作 |
|---|---|
| 基础 | 明确核心作用，建立读者认知 |
| 发展 | 增加新用法，核心作用不变 |
| 成熟 | 与世界观深度结合，可与其他系统联动 |
| 升华 | 作用对象从个人扩到世界/天道层级（需足够伏笔） |

**升级差异化三维度**（升级前后三维都无差异 → 读者无升级快感）：能力差异化（每级有质变新能力）/ 待遇差异化（不同等级不同社会待遇）/ 人际关系网差异化（不同等级面对不同层次的人）。挂 `power_system`——阶位表不只写战力数字，每阶补这三维标签。

**金手指骨相三型**（脑洞文卖点）：条件触发型（满足 X→得 Y，摸尸得宝/签到）/ 多条件阶段型（集齐碎片→阶段升级）/ 逆向型（失去 X→得更强 Y，越惨越强）。⚠️ 等级数量/升级节奏均 ⚠️经验值可调，写进 `power_system` 后由 `state_check.py` 校验单调，不靠模型记。

### 12.8 拉长剧情：设门槛 + 三层套娃

- **设门槛**（围绕核心卖点，脱离金手指/脑洞的门槛=无效剧情）：资源型 / 成就型 / 多条件型 / 动态门槛（主角超标就提高）/ 收集型。门槛分批提出，每跨一个立刻设下一个。
- **三层套娃同时运行**：大套娃（全书主线，对 `contract` 主线锚点）/ 中套娃（每卷阶段目标，对卷 `mainline_beats`）/ 小套娃（单元即时目标，对章纲）。配合"两长一短"期待：1 个短期（当前单元唯一明确目标）+ 1-2 个长期（远期/悬念/组织/人物）。

## 13. 品类专项 checklist：复仇打脸 + 女频爱情线（借鉴 oh-story-claudecode）

> 取材：oh-story-claudecode `genre-writing-formulas.md` + `genre-catalog.md`（MIT）。**定位**：§3（都市打脸）/§8（女频）已给六件套；本节是这两类**最易写崩处**的工艺细化 checklist，挂 `behavior_anchors`/`cognition` + `foreshadow-ledger` + `emotion-debt` + 毒点门，**不改其六件套**。

### 13.1 复仇 · 打脸文（主角冷静度 + 证据链）

**① 主角冷静度——落到 `behavior_anchors`**：主角永不失态，**用动作替代情绪**，对方越歇斯底里主角越平静。把"标志性冷静动作"钉进人物卡 `behavior_anchors`（具体事件非形容词），reviewer 验人物一致性（§04 `character_consistent`）。⚠️ 动作库是**示例**，照搬有 AI 味（→§09 桶1），写作时改写化用：

| 标志性动作（示例库 ⚠️可改写） | 效果 |
|---|---|
| 端起水杯，喝了一口 | 冷静到可怕 |
| 理了理西装并不存在的褶皱 | 优雅地宣判 |
| 转着车钥匙悠悠往里走 | 胜利者的从容 |
| 慢慢蹲下与对方平视 | 居高临下的慈悲 |

**② 证据链——落到 `foreshadow-ledger`**：每条打脸用的证据**必须前文埋过**（防天降），登记进伏笔台账（埋设章 + 回收章 + `status: open→微回应→closed`），逐层释放（一张一张甩，不一次倒完）。"公平线索没埋/证据天降" = §6 悬疑同款崩点；台账确保"埋了 + 按计划回收"闭环（§08）。

**③ 公开审判七步场景**（打脸名场面骨架）：设"竞技场"（股东会/家族寿宴/大堂，**必须公开**）→ 反派先"赢"（当众羞辱主角）→ 主角标志性冷静（动作库）→ 逐层揭露证据 → 反派逐级崩溃（得意→慌张→绝望→崩溃）→ 标志性驱逐台词 → 背影离场（全场死寂中头也不回）。**驱逐台词示例**（⚠️同样需化用："与狗，不得入内" / "从今天起她连大门都不能再踏进半步" / "厚重隔音门彻底关上，将两个世界永远隔绝"）。

> 对齐情绪债与钩子：挑衅 = 开 `emotion_debt`（积 `intensity`）；碾压/揭露 = `release`，受 `release_intensity >= intensity` 约束；四部曲（挑衅→隐忍→反转→余震）与 4 型钩子的完整机械见 §07，本节不重复。

**复仇打脸 checklist**：
- [ ] 主角"标志性冷静动作"写进 `behavior_anchors`，全程不失态（动作替代情绪）
- [ ] 每条打脸证据在 `foreshadow-ledger` 有"埋设→回收"记录，无天降证据
- [ ] 打脸节奏**不均匀**（反派嚣张度逐级递增；非每次等强度——§3 都市同款毒点）
- [ ] 公开审判场景"公开"且"反派先赢再崩"，证据逐层释放
- [ ] 受辱后约 2-3 单元内必反击（⚠️可调，按平台密度→§06），打脸后干脆利落不拖泥带水

### 13.2 女频爱情线工艺（男女频差异 / 四阶段 / CP 行为分级）

**① 男女频差异表**（写法基线，作 `config.tone` 维度选定）：

| 指标 | 女频 | 男频 |
|---|---|---|
| 情绪触发密度 | 3.23 次/千字 ⚠️经验值可调 | 2.50 次/千字 ⚠️经验值可调 |
| 反转密度 | 1.77 次/千字 ⚠️经验值可调 | 1.21 次/千字 ⚠️经验值可调 |
| 反转类型 | 信息差型（心声/真相） | 力量型（身份/实力） |
| 反派类型 | 渣男+绿茶+偏心父母 | 兄弟+老板+势利眼 |
| 复仇方式 | 信息差式（真相揭露） | 力量式（实力碾压） |

> 男频"称呼改变 = 身份揭露 + 集体震惊"；女频"公开审判式打脸 + 心声制造信息差"。这些密度数都是**经验值**，作 `sao_schedule` 排布时的参考量级、非硬编。

**② 爱情线四/五阶段**（按卷/arc 走，非全书 %）：相遇（被某特质吸引）→ 追求（用身份/财富制造便利，闺蜜震惊）→ 确认（女主心动走到一起）→ 阻力（家族/前任/商业对手，不能太强否则跑偏成虐）→ 化解（共克难关，感情升华）。每阶段的"感情推进"作为一种 `sao_payoff` type 登记进 `sao_schedule`。

**③ CP 行为分级 + 好感度门槛**（半编码——把"最软的爽点"加一道防线，治"无铺垫直接 CP"毒点）：给 CP 关系一个 `favor`（好感度）标量（进 relationship state），重大亲密行为按阶梯设门槛，**跨门槛才能发生**：

| favor 区间 ⚠️可调 | 允许的 CP 行为 |
|---|---|
| < 20 戒备/敌意 | 误会、冲突、嘴硬 |
| 20-40 心动萌芽 | 注意、不经意维护 |
| 40-60 暧昧拉扯 | 试探、吃醋、肢体微接触（扶/挡） |
| 60-80 确认关系 | 告白、牵手、第一次亲密 |
| 80+ 深度羁绊 | 共患难、托付、生死与共 |

> **裁决细化**：相邻两次感情推进章 `favor` 跃升过大（跳级）= "无铺垫直接 CP"毒点（§8），可做 semi-auto warn；但**推进力度仍由人在章纲标，AI 填空**——感情推拉火候是 §8 标注的"AI 最易崩点①、human-only 高软度部分"，门槛只防硬跳级，不替人定火候。门槛值按平台调（晋江三观+感情线审核最严、番茄宽，→§06）。

**女频爱情线 checklist**：
- [ ] 已按 `config.tone` 男/女频选定情绪/反转密度量级（⚠️经验值）与反转类型（信息差 vs 力量）
- [ ] 爱情线四/五阶段按**卷/arc** 排（非全书 %），每阶段感情推进登记进 `sao_schedule`
- [ ] CP 重大行为有 `favor` 门槛，无跳级（无铺垫直接 CP）；推进力度人工在章纲标
- [ ] 重生/穿越"已知未来"登记为可见性条目（`cognition.knows` + `visible_from_volume`），无信息差穿帮（§8）
- [ ] 双洁/三观红线按平台从严（→§06 晋江侧 + §1 内容红线）；圣母/白莲花女主入毒点黑名单

### 13.3 与现有品类块的关系

本节是 §3（都市赘婿/战神/系统打脸）与 §8（女频言情/古言/宅斗）六件套的**写法扩展**，复用其 `sao_engine`/`locked_reveals`/`forbidden` 字段，不平行造新框架：复仇打脸的"隐藏身份/真实实力"仍走 §3 的 `locked_reveals` 可见性截断；女频"重生信息差"仍走 §8 的 `cognition` 边界；`favor` 门槛与冷静 `behavior_anchors` 只是给这两类补的两个**可半检的细化裁决**。
