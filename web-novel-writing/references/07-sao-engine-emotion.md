# 07 · 爽点引擎与情绪债

> Purpose: 把"爽点"从写好人物的玄学，变成有因果链、可排布、可机检的工程对象。**用在步骤2（outliner 写章纲：给每章填 `sao_payoff` + `ending_hook` + 登记/兑现情绪债）和步骤6（reviewer 审爽点：对照本章排布验"需求建立→压抑→释放"链是否到位）。** 对应状态文档 `templates/emotion-debt.yaml`，对应章纲字段 `sao_payoff` / `reader_emotion_curve` / `ending_hook`（`templates/chapter-outline-template.json`）。

---

## 0. 一句话立场

网文的目标函数是**留存/追读/完读**——一套被平台算法直接计价的工程指标，不是文学的"质感回味"。所以爽点不能交给 AI 自由发挥（AI 默认匀速铺开、不会主动制造高潮低谷），必须**人类在章纲层显式标注爽点级别/情绪债/钩子骨架，AI 在约束内填空**。本文给的就是这层骨架的编码规则。

---

## 1. 原理一：爽点 = 安全感的社会延伸 + 期待感

爽点不是"写得好"，而是**激活读者一种被压抑后释放的社会性情绪**。底层拆成两条独立通道：

**通道 A · 安全感的社会延伸**（要铺垫、要对比、要兑现）

| 安全感类型 | 对应爽点手法 | 编码要点 | 反例（写崩） |
|---|---|---|---|
| **尊重感**（地位被承认） | **打脸**（地位反转） | 必须**先有需求/先被打压**——打脸需要铺垫，无铺垫的打脸=爽点免费 | 主角一出场就被跪舔，没有"被瞧不起"的债务可还 |
| **优越感**（碾压性的强） | **装逼**（实力/见识/格局碾压） | **对比角色越强越多，效果越好**——靠围观者衬托，不靠主角自夸 | 主角空喊"我很强"，没有强者陪衬当量尺 |
| **认同感**（价值观/选择被认同） | 主角主动选择被读者认同（仗义、护短、不忘本） | 用动作展示选择，不写心声总结 | 旁白直接判定"他是个好人" |

> 依据：龙空精华帖"对爽点的系统分析"——"尊重感代表地位→打脸需要铺垫先有需求；优越感→装逼，对比角色越强越多效果越好；认同感"[qidianclub/1326]。林海听涛（竞技文大神）补一刀实操：**"升级是小爽点、不要太多笔墨，读者期待的是升级后打脸；打脸要拉长高潮，用围观者评头论足充分利用高潮"**[ady101]。

**通道 B · 期待感**（条件反射，独立于安全感）

期待感来源于**巴甫洛夫条件反射**——读者被反复训练"看到 X 就预期会爽"（如系统提示音=要升级、敌人挑衅=要打脸）。它是**独立爽点**，可单独制造，且是章末钩子的心理基础（见 §6）。

> 依据：龙空帖"期待感来源于巴甫洛夫条件反射"[qidianclub/1326]；wangwen666 装逼打脸教程同源[wangwen666/post44]。

**可复用要点**：每个 `sao_payoff.type` 都要能回答"它激活哪条通道、靠什么对比/铺垫兑现"。填不出对比物或铺垫的爽点，就是 §7 的"爽点免费"反模式。

## 2. 原理二：情绪守恒 / 压抑-释放

爽点的节奏核心是一条守恒律：**释放强度必须 ≥ 已积累的憋屈债务**。

- **情绪守恒定律**："憋屈时长 / 强度，必须与释放（打脸/翻盘）力度成正比"[vocus/685aaa]。压得越狠、越久，打脸才越爽；压一下就翻盘=不解气，压很久只给短暂释放=被骗感。
- **先打一巴掌再给甜枣**：先降期待（让主角吃瘪/读者替他憋屈），再超预期解决——直接给糖不如先抽一鞭子。这是"压抑→释放"曲线的标准开法[wangwen666/post44]。

把它落成可检的不变量（情绪债账本强制，**不靠模型自觉**）：

```
release_intensity  ≥  intensity            # 释放够不够解气
duration_chapters  与  release_intensity 成正比   # 憋越久，还得越重
```

> ⚠️ 这是 §0"约束即质量"的体现：守恒律必须由 `emotion-debt.yaml` 自检 + reviewer 验，写进 prompt 让 LLM"记住"= 会漂移的伪约束。

## 3. 情绪债账本怎么用（对应 `templates/emotion-debt.yaml`）

账本是 AX 原框架缺失、必补的一层。它把每条"憋屈/打压/误会/危机"记成一笔**债**，强制后续有对应释放——这样"虐点无补偿"和"爽点免费"两类崩坏就能被代码扫出来，而不是靠手感。

**账本两张表（字段直接对齐模板，勿改名）：**

`emotion_debts`（每笔憋屈一条）

| 字段 | 含义 | 步骤2 登记时 | 步骤6 审核时 |
|---|---|---|---|
| `debt_id` | 债务 ID（D1…） | 新憋屈出现即开一笔 | — |
| `text` | 什么憋屈/打压/误会/危机 | 人/半自动填 | — |
| `intensity` | 憋屈强度 1-10 | 人定（节奏拍板是 human-only） | — |
| `incurred_ch` / `duration_chapters` | 第几章积累 / 已压抑多少章 | 登记 | 查"久未补偿" |
| `release_ch` | 计划第几章释放 | 排布时定 | 到期未释放→预警 |
| `release_intensity` | 释放强度（**必须 ≥ intensity**） | 排布时定 | `< intensity`→"不够解气" |
| `released` | 是否已还 | 默认 false | 释放章过后由 state-updater 翻 true |

`sao_schedule`（爽点排布，可机检密度）

| 字段 | 含义 | 取值 |
|---|---|---|
| `chapter` | 第几章 | — |
| `type` | 爽点类型 | 打脸 / 装逼 / 升级 / 扮猪吃虎 / 真相揭晓 / 情感兑现 … |
| `level` | 爽点级别 | 小爽 / 中爽 / 大爽（见 §4 刻度） |
| `setup_chain` | 因果链一句话 | "需求建立→压抑→释放"（验信息差→期待→兑现） |

**三条自检（driver / `state_check.py` 每卷跑一遍）：**

1. 连续 N 章无任何 `sao_schedule` 条目 → **"平路过长"预警**（见 §4 阈值）。
2. 存在 `released=false` 且 `duration_chapters` 过大 → **"虐点久未补偿"预警**。
3. 某条 `release_intensity < intensity` → **"释放不够解气"预警**。

**用法分工**：账本的"埋什么债/何时还/力度多大"是 semi-auto（人在卷/arc 边界排布），**登记、状态机流转、临近释放章提醒是 automate**（纯机械）。草稿只读账本，定稿才由 state-updater 抽 delta 回写（`released` 翻转、新债登记）。

## 4. 爽点密度刻度

把"爽点密度"做成可校验约束：每章章纲必须标注本章爽点级别，跨章统计密度，自动预警"连续 N 章无情绪高峰"。

| 刻度 | 工业经验值 | 来源/置信度 |
|---|---|---|
| **小爽** | 每章 ≥ 1 个（升级、装逼、一次小打脸、一个信息差兑现） | 马良写作 [high] |
| **中爽** | 每 3-5 章 1 个（**完整打脸闭环**：铺垫压制→反转碾压→围观震惊） | 马良写作 [high] |
| **大爽** | 每卷 1 个（阶段 Boss / 换地图级的总爆发） | snowflake-fiction [high] |
| **"平路"上限** | 连续 **3 章**无情绪高峰 → 追读开始掉 | 马良写作 [medium] |
| **情绪高峰间隔（参考线）** | 追读 top10% 作品约 **每 1.8 章**一次；bottom10% 约 4.7 章 | ⚠️ **单一来源（马良写作），medium，当可调参数不当真值** |
| **更粗的工业刻度** | 每 3 章小爽 / 每 10 章中爽 / 每 30 章大爽（阶段Boss）/ 每 100 章超大爽（换地图） | snowflake-fiction，与上面口径不同，并存参考 |

> ⚠️ 1.8 / 4.7 章是单一来源数据，平台算法不公开；写进 skill 当**可调参数**，按自有追读数据校准，绝不硬编成"真值"。

**可复用要点**：密度是结构层、人定骨架；AI 只负责"按本章标注的级别把爽点写到位"。两套刻度（马良细 vs snowflake粗）选一套挂载，别混用导致预警阈值打架。

## 5. 打脸四部曲 / 装逼三层（爽点的两个标准模板）

爽点不是一个动作，是一段**结构**。两个最高频的可模板化结构：

**打脸四部曲**（都市/系统/无敌流的单元节拍，对应 `sao_payoff.type=打脸`）

| 段 | 名称 | 写什么 | 对应情绪债 |
|---|---|---|---|
| 1 | **挑衅** | 反派/路人当面打压、瞧不起、立 flag | 开一笔 `emotion_debt`（积累 intensity） |
| 2 | **隐忍** | 主角扮猪/按住不发，债务持续累积 | `duration_chapters` 增长 |
| 3 | **反转** | 身份/实力直球揭穿，**把话说死**、不绕潜台词 | 释放（`release_intensity ≥ intensity`） |
| 4 | **余震** | 围观者评头论足、反派破防、连锁震惊——**拉长高潮充分利用** | 把释放写够本，防"爽点一笔带过" |

> 依据：wangwen666《脑洞爽文5大技法》打脸四部曲"挑衅→隐忍→反转→余震"，并给各平台周期参数（番茄 5-8 章 / 起点 15-20 章 / 飞卢 3 章一个打脸周期）[wangwen666/post186]；"余震拉长"=林海听涛"打脸要拉长高潮、用围观者充分利用高潮"[ady101]。

**装逼三层级**（强度梯度，对应 `sao_payoff.type=装逼`）

| 层 | 名称 | 手法 | 适用 |
|---|---|---|---|
| 1 | **基础装** | 主角直接秀实力/见识 | 日常小爽，密度填充 |
| 2 | **温酒斩华雄式·侧面压** | 不正面写主角多强，写**旁人反应/对比**衬出强（关羽温酒斩华雄，镜头在帐内不在阵前） | 中爽，最高级、最不油腻 |
| 3 | **狠压式** | 先被极致打压/绝境，再碾压翻盘 | 大爽，配深债务（§2 守恒律） |

> 依据：wangwen666 新媒体爽点教程把装逼分三层（基础装 / 温酒斩华雄式侧面压 / 狠压式）+ "先打一巴掌再给甜枣"[wangwen666/post44]。

**可复用要点**：四部曲/三层是 AI 最适合"按字段填充"的结构——人在章纲定哪一层、配多深的债，writer 填具体台词与围观反应。**反模式见 §7"压点写串"**：愤怒式打脸（你侮辱我→我打回去）和瞧不起式打脸（你看不起我→我证明给你看）的情绪基调不能混用，混了就泄气。

## 6. 章末钩子 4 型 + 怎么在章纲提前规划

**原理**：读者决定看不看下一章，就看这章**最后三行**。钩子是追读命门，且必须在章纲里**提前规划**——不能写完正文再想结尾（写完再想 = 自然收束 = 帮读者按退出键）。钩子的心理基础是 §1 的期待感（条件反射制造好奇缺口）。

**4 型钩子**（对应章纲 `ending_hook.type`，取值严格用这四个）：

| 型 | `ending_hook.type` | 触发的读者心理 | 例 |
|---|---|---|---|
| 悬念式 | `悬念式` | "接下来会怎样？" | 门被推开，门口站着不该出现的人 |
| 反转式 | `反转式` | "原来如此？！" | 主角的盟友其实是幕后黑手（注意 §防剧透：别提前泄底） |
| 情绪炸弹式 | `情绪炸弹式` | 愤怒/憋屈/燃，强情绪驱动翻页 | 至亲被当众羞辱，主角攥拳——债务开在章末 |
| 信息投放式 | `信息投放式` | "这条新信息意味着什么？" | 抛出一个改变格局的设定/数字/名字 |

**怎么在章纲提前规划（步骤2）：**

1. 章纲 `ending_hook.text` 字段是**最后填、但生成前必须填满**的——它是 writer 的硬约束，不是事后润色。
2. 钩子要**跨章成链**：`next_chapter_sketch.must_setup_for_next` 记本章为下一章铺了什么；下一章 `chapter_purpose` 要能接住这个钩子（钩子有处可落，不是空挖坑）。
3. **章节名 = 读者承诺层**：章名要既提高续读意愿、又被本章内容兑现（标题党挖坑不还=毒点）。
4. 钩子是 semi-auto：型号/内容人定或按"跨章问题链"规则定，AI 按字段生成最后三行；reviewer 抽检兑现度（review-report 里 `ending_hook` 项 max 15 / min 10，**钩子不成立直接低于最低线 → REVISE**）。

> 依据：马良写作"读者决定要不要看下一章就看你这章最后三行"，四型钩子模板 + "钩子要在蓝图里提前规划"[maliangwriter]；webnovel-handbook 把章末钩子升级为跨章"读者问题链/好奇心缺口"管理 + "章节名=读者承诺层"专项[miserylee/webnovel-handbook]。

## 7. 反模式（爽点/情绪债写崩的三种死法）

| 反模式 | 症状 | 根因 | 账本怎么扫出来 |
|---|---|---|---|
| **虐点无补偿** | 一直憋屈，迟迟不还、或永不还 | 开了债不记账，写着写着忘了 | `released=false` + `duration_chapters` 过大 → 预警 |
| **爽点免费**（无铺垫直接爽） | 一上来开挂碾压，读者没"等到"的快感 | 跳过 §2 压抑段，没债就还 | `setup_chain` 填不出"需求建立→压抑"前两环 → 标记 |
| **压点写串** | 打脸泄气、不解气 | 愤怒式打脸 vs 瞧不起式打脸混用；或 `release_intensity < intensity` | 守恒律自检 + reviewer 看情绪基调一致性 |

**另外两条节奏级反模式（来自三桶反 AI 味·桶3）：**

- **平均化节奏**：AI 默认每段差不多长、每事件差不多详 → 读起来寡淡。不在章纲显式标加速/减速/急停，节奏必"太平"。`sao_schedule` 连续空 N 章 = 平路预警。
- **升级喧宾夺主**：把"升级"本身写太多笔墨当大爽 → 读者真正期待的是**升级后的打脸**。升级是小爽点、点到为止，把笔墨留给兑现（林海听涛）[ady101]。

> ⚠️ 本文档自身也不许违反三桶清单：上面所有"原理"段都是直陈操作规则、无华美空洞、无匀速铺陈——这正是网文要的"直给"。

## 8. 与生产循环的接口（落到哪两步）

| 步骤 | 谁 | 用本文什么 | 产物字段 |
|---|---|---|---|
| **步骤2 · outliner（写章纲）** | semi-auto，人定级别/债 | §1 因果链选 type、§4 刻度定 level、§5 四部曲/三层选结构、§6 钩子型号 | 章纲 `sao_payoff{type,level,setup_chain}`、`reader_emotion_curve`、`ending_hook{type,text}`；账本 `emotion_debts` 开/还、`sao_schedule` 排布 |
| **步骤6 · reviewer（审爽点）** | automate，独立上下文 | §2 守恒律、§3 三条自检、§6 钩子兑现 | review-report `weighted_scores.sao_payoff`(max20/min14) + `ending_hook`(max15/min10)；任一低于 min → REVISE |
| **每卷维护 · state_check** | automate | §3 三条自检 + §4 平路阈值 | 三类预警（平路过长 / 久未补偿 / 不够解气） |

**精确化边界**：爽点的**走向与力度拍板是 human-only**（憋屈强度、释放时机、是否到位的判断）；**编排填充是 semi-auto**（机制可编码，AI 按账本/章纲字段填）；**密度统计、债务状态流转、临近释放提醒是 automate**（纯机械）。

## 9. 章首钩子 7 式（借鉴 oh-story-claudecode）

**原理**：§6 的钩子4型管的是"看不看**下一**章"（章尾三行）；章首钩子管的是"打开**这**章后，前 100 字内愿不愿继续读下去"。两者是一对——章尾负责把读者推过翻页动作，章首负责接住、不让他在开头风景描写里流失。**黄金三章**与日更完读率对前 100 字尤其敏感（⚠️经验值，可调）。

**7 式**（与 §6 章尾4型互补；章尾是强制 typed 契约，章首是可选写法标注，见下）：

| # | 式 | 公式 | 例 | 适用开篇策略 |
|---|---|---|---|---|
| 1 | 悬念对话 | 直接从意味深长的对话切入 | "你确定要这么做？""确定。""那你别后悔。" | 制造好奇 |
| 2 | 闪前碎片 | 先抛结果碎片，再回正常叙事 | "后来他才知道，那个电话改变了一切。但此刻他一无所知。" | 制造好奇 |
| 3 | 倒计时 | 开头即建立时间压 | "距离合同到期，还有 72 小时。" | 直接冲击 |
| 4 | 神秘独白 | 第一人称诡异内心独白 | "我一直在想，那天若没回头，会不会不一样。" | 制造好奇 |
| 5 | 反差场景 | 两个截然不同的场景并列 | "一边婚礼香槟塔堆得老高，一边医院走廊的灯在闪。" | 对比冲击 |
| 6 | 未完成动作 | 动作进行中被截断 | "他刚把钥匙插进锁孔，门里传来一声轻响。" | 直接冲击 |
| 7 | 意象预示 | 用环境意象暗示将发生的事 | "天边烧得通红，像什么东西在着。她抬头看一眼，继续走。" | 对比冲击 |

**决策路由（开篇策略 → 式号）**：直接冲击 → 6 / 3；制造好奇 → 1 / 2 / 4；对比冲击 → 5 / 7。

**怎么挂在章纲（步骤2，可选标注，不动 §6 契约）**：

- 章纲可加**可选**字段 `opening_hook`（标 `{type, text}` 或只标 type）。**它与 §6 的 `ending_hook` 区别明确**：`ending_hook` 是**强制 typed 契约**（4型枚举、reviewer 必查、低于 min 线即 REVISE）；`opening_hook` 是**可选写法标注**，不进强制评分，缺省即可。这样既给黄金三章一个抓手，又不给中期日常章添不必要的枷锁。
- **何时必标**：第1章 + 黄金三章（§4 阶段定位"必须强"），以及高潮前的爆发章；中期日常章可空。
- **跨章接力**：本章 `opening_hook` 优先**接住上一章的 `ending_hook`**——上一章章尾留"门被推开，站着不该出现的人"，本章章首就用悬念对话/未完成动作直接落地，形成"尾钩 → 首钩"闭环，而非另起炉灶（呼应 §6 跨章问题链）。

**反模式**：风景/天气流水账开头（前 100 字无钩子）；连续两章用同一式（钝化）。reviewer 抽检：章首前 100 字是否落在 7 式之一、连续两章是否撞型。

> 依据：oh-story-claudecode `hooks-chapter.md`（章首7式 + 章尾13式 + "前100字定生死" + "连续两章不用同一种钩子"）。

## 10. 情绪弧线与期待感管理（借鉴 oh-story-claudecode）

**原理与边界**：§1-2 给的是**单个爽点**的因果链（点）；本节给的是**一卷/一个 arc 的情绪形状**（线）。爽点是点，弧线是线——弧线决定这些点怎么排成有起伏的曲线。挂在章纲 `reader_emotion_curve` + **卷/arc 规划层**之上。

**⚠️ ADAPT（长篇连载关键）**：oh-story 的弧线 % 是【单篇短篇全文】的位置。长篇连载没有"全书 60%"这种有意义的拐点——**所有转折点 % 一律读作"所在卷 / arc 进度的相对位置"**，不是全书进度。长篇 = 多条弧线**串联 + 叠加**（每卷一条主弧，arc 内可嵌小弧）。单卷/arc 内可叠加 ≤ 2 种弧线，别更多。

**6 种弧线**（卷/arc 内的情绪形状，填进 `reader_emotion_curve`）：

| 弧线 | 卷/arc 内转折位（⚠️经验值，可调） | 写什么 | 网文典型 |
|---|---|---|---|
| **V 形**（虐 → 治愈） | 谷底 60-70%，治愈急升，结尾高于起点 | 谷底够深、转折突然、治愈不拖 | 重生复仇、破镜重圆 |
| **倒 V**（爽 → 反转 → 低谷） | 高点 40-50%，坠落快于上升，BE 不加糖 | 男频少用，女频虐恋/BE 向 | BE 虐恋、信任崩塌 |
| **W 形**（多起伏） | 每峰高于前峰，末峰 80-90%，上限 3 起伏 | 峰间留喘息 | 追妻火葬场、商战博弈 |
| **递进**（台阶上升） | 无回落，最高 85-95%，中途只给小胜 | 一个比一个狠，不提前释放 | 打脸爽文主轴、逆袭翻盘 |
| **延迟满足**（长铺垫 → 爆发） | 前 70-80% 压制/铺垫，末 20-30% 集中爆发后快收 | 铺垫段每段要有信息量、别无聊 | 扮猪吃虎、身份揭秘、长线布局 |
| **急转**（70-80% 急转弯） | 急转只占 1-2 段，前埋线索回看须说通 | 接 §反转·认知反转 | 悬疑、细思极恐、动机反转 |

> 决策：虐完翻盘 → V；甜完刀 → 倒V；多反转 → W；层层打脸 → 递进；隐忍后爆发 → 延迟满足；结尾颠覆 → 急转。

**确定型 vs 不确定型期待**（决定 `sao_schedule` 怎么排）：

- **确定型（爽文）**：读者**知道主角会赢**，看点是"**怎么**赢"——过程创意 > 结果悬念。**爽文不怕"剧透"，因为期待在过程不在结果。** → 排爽点时把创意/落差放过程，结果可前置（甚至底牌前置，见 §11 舞台）。
- **不确定型（悬疑）**：读者**不知道结果**——结果悬念 > 过程体验。 → 护住结果、别提前泄底（呼应伏笔台账"埋伏笔不泄底"）。
- 同一长篇可主线确定型（升级打脸）+ 支线不确定型（身世谜题）并行。

**三层情绪分离**（writer 写释放/虐点时的裁决细化，防平面化）：同一场景拆三层——① 角色自己的情绪 ② 文本传递的情绪 ③ 读者实际感受的情绪，**三层可以完全不同**。例：女主当众被退婚、强忍着还挂着笑——角色：屈辱；文本：撑；读者：爽前蓄力。这条挂在 emotion-debt 的**释放写法**上：登记一笔虐点债时，writer 不必让"角色哭 → 读者也只能跟着哭"，而是用三层错位把虐点写成读者的"蓄力快感"。

**期待感六法则**（卷/arc 内排期待用）：

1. **期待最大化**：爽点到来前一刻张力最高（将满未满，需求最大）——别把这一刻一笔带过。
2. **信息顺序操纵**：先植入什么，读者先往哪想；控制信息顺序 = 控制预期。
3. **期待递增**：在基底期待上叠具体细节缺口，细节须与核心梗一致。
4. **断期待禁止**：旧期待兑现前，新期待种子必须已下——否则读者读完即走（连接下方多线悬念）。
5. **下行安全原则**：写情绪下行（虐/挫折）时，**锅都是别人的，功都是主角的**；下行中必须给安全感（可能的解法/潜在收获）。→ 挂 emotion-debt：登记虐点债时账本旁注"安全感来源"，否则虐点 = 真憋屈 = 弃书（区别于 §7"虐点无补偿"，这条是"虐点无安全垫"）。
6. **爽点递增对比**：连续爽点逐级递增，三维度——影响范围（个人 → 群体 → 社会）、揭示深度（表象 → 本质 → 颠覆）、身份落差（路人 → 大佬 → 全场震惊）。→ 与 §4 爽点级别刻度对齐：级别升 = 至少一个维度升。

**多线悬念周期**（结构层不变量）：

| 弧 | 跨度（⚠️经验值，可调） | 挂在哪 |
|---|---|---|
| 短弧 | 2-3 章（一场冲突/打斗） | `sao_schedule` 单元 |
| 中弧 | 5-8 章（一段关系变化/小谜题） | arc 级 |
| 长弧 | 整卷（终极秘密/主线大反转） | **主线锚点 + 伏笔台账** |

**可机检不变量**：规划时三弧同铺，**任意时刻 ≥ 2 条悬念线在跑**。这是类似 §3 的自检——`state_check` 每卷扫"是否存在 < 2 条未闭合悬念线" → **"期待断裂"预警**（呼应法则4）。

**与生产循环接口**：步骤2 outliner 在**卷/arc 边界**定弧线型 + 填 `reader_emotion_curve`、铺三层悬念；步骤6 reviewer 验"调动-释放是否配平 / 有无只调动不释放 / 断期待"；`state_check` 验 ≥ 2 悬念线 + 弧线转折位是否落在卷/arc 合理区间。

> 依据：oh-story-claudecode `emotional-arc-design.md`（六弧线 + 期待六法则 + 三层情绪分离 + 确定/不确定期待）、`hooks-suspense.md`（多线悬念周期 + 强度5级 + 期待接力）。

## 11. 二级结构化装逼 + 震惊分层（借鉴 oh-story-claudecode）

**原理与边界**：§5 给的是装逼**三层强度梯度**（基础/温酒斩华雄式/狠压式）；本节给的是装逼的**社会传播结构**——一次装逼如何在"在场人群"里扩散、并升级成后续剧情的钩子。挂在 §5 之上当写法细化，对应 `sao_payoff.type=装逼`。

**一级 vs 二级装逼**：

- **一级装逼**：主角展示 → 旁观者震惊（"他好强"）。情绪满足，但**用完即弃**。
- **二级（结构化）装逼**：主角的展示不只震惊旁观者，**还直接波及在场某个角色的核心利益/计划/认知** → 震惊从"好厉害"变成"**这跟我有关系**"。
  - 主角的成果正好是 A 苦求的东西 → A 焦虑升级
  - 主角的实力打破 B 的认知框架 → B 必须重新规划
  - 主角的存在威胁 C 的地位 → C 生敌意或转投靠

**两条契约对齐**（这是二级装逼不沦为空话的关键）：

1. **反应基于人物卡，不是雷同模板**：每个在场角色的震惊**必须基于其自身利益 + `cognition`**，而非统一"倒吸一口凉气"。→ 直接挂人物卡 `behavior_anchors + cognition`：reviewer 验"在场配角反应是否各自从其认知/利益出发，有无千人一面的雷同震惊"。
2. **每次装逼至少波及一个角色核心利益 → 产钩子**：二级效果创造新矛盾/关系变化 → **登记进 `sao_schedule`（后续爽点种子）或伏笔台账（后续剧情线）**。这把装逼从"一次性爽点"升级成"剧情发动机"。

**震惊传递链**（先铺人际关系，否则没有传递通道）：

```
最内层（贴身崇拜：丫鬟/亲信）
  → 中间层（同场组织者、才子、对手震惊）
  → 核心层（大佬膜拜、高位者震惊）
  → 反向回传（核心层反应传回群众层，二次震惊）
```

装逼是**正向（由下往上）+ 反向（大佬震惊传回群众）**双向的。高潮后众人态度转变 = 情绪闭环。

**舞台两维：大小 × 高度**（每次装逼自检"还能不能更大/更高"）：

- **大小 = 观众数量**：室友看 → 直播间万人 → 全赛区 → 全世界
- **高度 = 对手身份 + 场合重要度**：普通排位 → 王者局 → S 赛决赛
- → **与 §4 爽点级别对齐**：小爽配小舞台、大爽配大舞台；`sao_payoff.level` 升级时，舞台大小/高度至少一维要跟上。不崩前提下，观众能拉多少拉多少。

**震惊三层**（从弱到强，配合舞台扩大）：

| 层 | 名 | 写法 | 后续价值 |
|---|---|---|---|
| 1 | **点震惊** | 一个人震惊一下（最弱） | 纯情绪，无后续 |
| 2 | **网震惊** | 关系网周围都有反应 | 态度变化 |
| 3 | **深度震惊** | 多层叠加：成就1震 → 成就2震 → 更厉害的成就3引爆 | 带信息展露 + 关系进展 + 拉期待 |

**关系网震惊层级**（选层依据 = 该角色**后续戏份**，有后续才有价值）：路人（弱）→ 熟人（中）→ 重要角色（强，带信息/关系/期待）→ 高位者（最强，侧拉读者对主角的期待）。

**道具可视化**（写"余震"——§5 打脸四部曲第4段——的具体手法）：用道具的物理变化承载震惊强度，**比直说"他震惊了"强一百倍**：

> 成就1：对方把椅子把手捏出一道裂痕（轻）→ 成就2：椅子满是裂纹（中）→ 成就3：捏爆椅子把手（强）

每次递进都给明确视觉/物理变化，别用"他大惊失色"这类干说。

**铁律：该爽不爽，比毒点还毒。** 放了底牌，反派就要受**对应压制**——该破防写破防、该羡慕写羡慕、该扭转战局写扭转，金手指对剧情的效果展示得清清楚楚；为情绪顺畅，甚至可在一定程度上牺牲合理性。**寸止/拉扯可以，但别让主角委屈**（被羞辱、放了底牌还挨揍 = 扶不起的废物 = 比发刀子更毒）。这条是 §2 守恒律的释放端落地：`release_intensity` 不够 = 该爽不爽。

**与生产循环 + 反模式**：

- 步骤2：装逼场景标 `舞台{大小,高度}` + 二级波及对象（进 `sao_schedule` / 伏笔台账）。
- 步骤6：reviewer 验——震惊是否分层（非雷同反应）/ 底牌是否有对应压制 / 主角是否委屈 / 道具是否可视化。
- 反模式：① 统一"倒吸凉气"（平面化，违契约1）② 放了底牌被反打（该爽不爽）③ 震惊只用路人（无后续 = 无价值）④ "他震惊了"干说不给道具。

> 依据：oh-story-claudecode `style-combat-face.md`（二级结构化装逼 + 震惊传递链 + 舞台大小×高度 + 该爽不爽比毒点还毒）、`hooks-suspense.md`（震惊三层点/网/深度 + 关系网层级 + 道具可视化）、`reversal-toolkit.md`（超额收获/情绪拉扯8步）。
