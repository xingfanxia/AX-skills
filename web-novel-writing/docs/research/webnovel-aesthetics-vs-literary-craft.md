# 网文原生「反AI味 + 质检」清单：文学/游戏craft铁律在中文网文里的批判性裁决

> 调研产出（Opus subagent, Tavily-backed, 2026-06-27）。回答用户的关键 pushback：文学/游戏的反AI味清单不能全量照搬到网文。结论：拆成三个桶——词句层全量复用、结构层改分层规则、新增网文专属轴。本文是 `web-novel-writing` skill 的审校 rubric 与反AI味模块的一手依据。

## A. 执行摘要

把文学/游戏的「反AI味清单」整体搬到网文，最大的风险是**用错了"轴"**。文学/游戏craft的隐含目标函数是"质感、回味、艺术完成度"；网文（尤其起点/番茄/七猫的连载爽文）的目标函数是**"留存率 / 追读率 / 完读率"——一套被平台算法直接计价的工程指标**（番茄、七猫靠广告完读率分钱，作者收益与读者"划走速度"直接挂钩）[wangwen666; woshipm]。两套审美在多个维度上**方向相反**：文学要"潜台词、克制、留白、show不tell"，爽文要"情绪锚点直给、装逼打脸把话说死、信息差极致、即时反馈"[woshipm; wangwen666/post44]。

但关键发现是：**文学清单不是"全错"，而是"对错混杂且高度分品类"**。粗暴归纳——①**句子/词汇层面的反AI味（紫色文风、堆形容词、翻译腔、节奏均匀）几乎可全量复用，而且正是网文编辑肉眼鉴AI的首要标准**[thepaper];②**结构/情绪层面的文学铁律（show-don't-tell、潜台词、避免信息倾倒、视角纪律）则需要"分层裁决"**，对"人设/逻辑"成立、对"情绪/爽点/卖点"恰恰反转;③**网文还有一整条文学清单完全没有的轴——爽点工程 + 毒点规避 + 黄金三章留存**，这条轴上不达标，文笔再"干净"也是死。最危险的单点误用是把"show-don't-tell"和"避免信息倾倒"教条化——它会让AI写出编辑一眼判死的"华美的空洞"（开篇大段环境描写、做加法不推进剧情）[thepaper]。

## C. 网文原生质检维度（文学清单里没有的轴）

**① 爽点工程（最核心，文学清单完全缺失）**
- **怎么查**：数"情绪高峰频率"——每章≥1个小爽点、每3-5章1个中爽点（完整打脸闭环：铺垫压制→反转碾压→围观震惊）、每卷1个大爽点[maliangwriter]。检查"压抑→释放"曲线是否到位（"压的越狠打脸越爽"[wangwen666/post44]）。
- **好长什么样**：爽点密度均匀分布，装逼/打脸/升级/扮猪吃虎有明确"信息差→期待感→兑现"链条；即时反馈（路人震惊脸、实力跃升）。
- **坏长什么样**：爽点被"无效日常"稀释；打脸"压点写串了"（愤怒式 vs 瞧不起式混用）；"小无敌不前置"——读者迟迟等不到承诺的爽。

**② 黄金三章 / 开篇留存**
- **怎么查**：前300字有无"爆点"（危机/金手指/反转）；500字内主角出场；第一章核心冲突浮出；金手指第三章前亮相；**章末有钩子**[wangwen666/204; snowflake-fiction]。免费平台(番茄/七猫/飞卢)阈值更严，付费平台(起点/纵横)可慢热。
- **好**：开篇即"建立情绪锚点"——三章内告诉读者"看这本书你能获得什么核心快感"[woshipm]。
- **坏**：从"我叫XX是个普通人"开场；穿越前铺垫三五万字后全丢；八百字主角没出场，读者不知道代入谁。

**③ 节奏与"水文 vs 干货"（文学清单完全没有）**
- **怎么查**："平路(日常/感情戏/休整)连续超过3章→追读开始掉"[maliangwriter]。注意网文允许一定"水"，毒在主线停滞。
- **好/坏**：读者眼中"好看、有期待感=干货；拖沓、主线停滞=水文，与字数无关"[sohu/234]。

**④ 毒点清单（分品类，劝退重灾区）**
- **男频通杀**：绿帽(第一毒点)/送女/主角弱智/反派降智/圣母/油腻种马/重要角色death/反三观/烂尾太监[sohu/234; wangwen666/194]。
- **分平台容忍度**：晋江对"圣母/三观"容忍最低、感情线必须有铺垫拉扯；飞卢**不能虐主**(主角得一路开挂见谁秒谁)、绿帽零容忍；起点容忍慢热但不容逻辑漏洞[sohu/516; wangwen666/194]。

**⑤ 代入感 / 爽文主角设计**
- **怎么查**：主角是否"让读者爽地代入"=掌控型、金手指赋予绝对比较优势[cul-studies]。
- **好**：可加**性格**瑕疵（好色/贪财/嘴贱/惜命，如白小纯、段德）增加记忆点和"人味"[aipiaxi; 163]。
- **坏**：**能力/智商**上的弱点(降智)、道德上的圣母——这是毒点不是弧线。**注意与文学冲突**：文学要"主角有缺点+成长弧线"，爽文要"主角强但有趣"，**玛丽苏/杰克苏在无敌流/苏爽文里是feature不是bug**[cul-studies; jjwxc]。

**⑥ 断章 / 章末钩子**
- **怎么查**：每章最后三行有无钩子（悬念式/反转式/情绪炸弹式/信息投放式）[maliangwriter]。
- **坏**：章节自然收束、情绪落地——等于"帮读者按了退出键"。

## D. 网文版「AI味」的精确定义 + 哪些能救 / 哪些是模型天花板

网文编辑（澎湃/骨朵访谈）总结的**AI味五大特征**，比通用"AI slop"清单更准[thepaper]：
1. **华美的空洞**——开篇堆环境描写、一句话形容词>3个、"显示器蓝光投下扭曲倒影，我数着第37根雨丝划过屏幕"（雨丝怎么划过屏幕）。这是编辑判AI的首要标准，因为"文笔差不会有那么多华美形容，文笔好又不会在开篇堆无意义描写"——AI暴露在这个矛盾点。
2. **逻辑bug**——人设前后矛盾（父母双亡却要回家继承家业）、公交车坐三天三夜。
3. **做加法破坏节奏**——"有经验作者往死里做减法，AI往死里做加法"；该紧张的掀盖头场景，AI塞大量新娘外貌描写。
4. **不主动推进剧情**——只会"环境描写+名词解释+遇到各路人马+心声"，看着写很多，剧情没推进。
5. **无法情感共鸣**——文字拼凑，"失了魂儿"。

**句子层的AI味=翻译腔**：骨架为英文的中文（主谓搭配、动词选择、段落节奏全是英文的），鲁迅/王小波骂过一百年，现在被AI批量生产[yage.ai]。这与"华丽辞藻硬拼"叠加（RLHF奖励正式/华丽词汇）[sina]。

**能靠prompt/风格样本救的**：
- 词汇黑名单（delve/realm/tapestry/"这不仅是X更是Y")、"破坏对称性/段落长短变化/禁列表多段落/大纲人定模型填空"[aitntnews; sina]。
- **节奏可控**：在章节蓝图里标注"本段慢/本章加快""紧张段短句≤15字"，AI会听[maliangwriter]。
- "往死里做减法"、章末钩子、爽点级别——都可写进结构化指令。
- "裁判模型"工作流（另开窗口当冷酷编辑挑AI slop再重写）[aitntnews]。

**模型预训练bias、prompt救不动的天花板（⚠️部分推断，confidence: medium）**：
- **"人味儿"/语感的手工感**——文法上微小的错位、多一个"最"字、微妙的不平衡，是"人是有个性、有感情"的非语言信息[YouTube文字诊疗室; xinwen.bjd]。模型天生抹平这些。
- **长程逻辑一致性 + 主动推进**——"把1-10章喂给AI写11章，它做不到"[xhby/跳舞]；不经精细调教生成逻辑自洽的大长篇，目前能力上限达不到[thepaper]。
- **网文特有的"梗"和语感**——炸天帮、"今日无事勾栏听曲"这类社区共生的梗，靠预训练语料给不出。
- **情绪的"压抑→释放"曲线主动设计**——AI天然匀速，需要人类在结构层强制注入。

> 实务结论：**AI味在网文里≈"翻译腔×华美空洞×节奏平×不推进剧情"**。前两项是词句层、可prompt救；后两项是结构层，需要人类先定爽点/节奏/钩子骨架，让AI填空——而非让AI当包工头。

## E. 《网文章节质检清单（反AI味 + 爽点 + 毒点）》草案

> 每条标注来源：【沿用文学】=文学反AI味清单可直接复用 / 【网文专属】=文学清单没有 / 【推翻×】=反转了文学清单第×条。建议每章过一遍，分区打分（每项0-2分：0=踩雷，1=及格，2=优秀）。

### 区块1 · 对白与情绪
- [ ] 角色对白有性格区分，不是"你好我好"背课文 【沿用文学：避免on-the-nose的"机器对白"】
- [ ] **爽点场景敢于把话说死、身份直球揭穿**（装逼/打脸不绕弯） 【推翻×2：潜台词>说死】
- [ ] 情绪点直给到位（"不爽→爽""愤怒→解气"压释对应正确，没写串压点） 【网文专属】
- [ ] 感情线/悬念保留必要潜台词与拉扯（晋江品类尤其） 【沿用文学】

### 区块2 · 爽点与节奏
- [ ] 本章≥1个小爽点；近3-5章有1个中爽点（完整打脸/升级闭环） 【网文专属】
- [ ] "压抑→释放"曲线到位，信息差服务于期待感 【网文专属】
- [ ] **章末有钩子**（悬念/反转/情绪炸弹/信息投放四选一） 【网文专属】
- [ ] 节奏不"匀速"：紧张段短句、平静段正常；近3章没有连续"平路" 【沿用文学(长短交替) + 反AI核心】
- [ ] 主线在推进，不是无效日常/感情戏喧宾夺主堆字数 【网文专属：水文毒点】

### 区块3 · 人设与代入
- [ ] 主角"强且有趣"，可有**性格**瑕疵(惜命/毒舌/贪财)增记忆点 【推翻×部分：玛丽苏在苏爽文是feature】
- [ ] **无降智/圣母/油腻种马/双商掉线**（能力·道德缺陷=毒点，非弧线） 【网文专属毒点；与文学"缺陷弧线"冲突】
- [ ] 主角是读者唯一明确的情感代入点，开篇没被配角抢戏 【网文专属】

### 区块4 · 世界观与信息投放
- [ ] 卖点/金手指/系统面板**前置直陈**（读者要的信息直给） 【推翻×3：信息倾倒在系统流是设计】
- [ ] 但**世界观lore不一股脑灌**（开篇不大段放设定/地图/多人物） 【沿用文学：避免info-dump】
- [ ] 金手指/战力规则前后自洽，无"机械降神/战力崩坏" 【沿用文学×8 + 网文毒点】

### 区块5 · 文笔与反AI味（词句层）
- [ ] **无"华美的空洞"**：开篇不堆环境描写，一句话形容词≤2个，一个强动词>三个修饰 【沿用文学×4 + 鉴AI第一标准】
- [ ] **无翻译腔**：句子骨架是中文（动作描述思考、体面现在分词、空悬动词"接住"=删） 【沿用文学 + 反AI核心】
- [ ] **品类适配**：白文(番茄/七猫)求"白爽"；古言/古典仙侠/虐文允许辞藻、文青、古气（此时辞藻=卖点） 【推翻×4(分品类)】
- [ ] 整体"做减法"，无AI式"做加法"堆砌；剧情>文笔 【网文专属(thepaper编辑判据)】

### 区块6 · 视角纪律
- [ ] **一章一POV**，不在同场戏跳头；无"上帝插嘴(殊不知/山的另一边…)" 【沿用文学×7：现代网文编辑强制】
- [ ] 长篇用第三人称限知；短篇爽文/情感可第一人称（字数匹配视角） 【网文专属(平台/字数规则)】

### 区块7 · 品类适配 / 平台红线
- [ ] 选对平台节奏阈值（免费站前300字爆点；付费站可慢热） 【网文专属】
- [ ] 避开平台三观红线（绿帽/送女/夹带私货；晋江审核最严） 【网文专属毒点】
- [ ] 题材专业性过关（历史无穿帮、专业文不"过家家"） 【网文专属】

**给 skill 设计者的落地建议**：不要把文学反AI味清单当"全集复用"，而是**拆成三个桶**——(1)词句层(紫色文风/翻译腔/形容词堆砌/节奏匀速)**全量复用且加权**，这是网文编辑鉴AI的第一道关；(2)结构层(show-don't-tell/信息倾倒/潜台词/视角)**改成"分层规则"**而非禁令——对人设/逻辑/世界观lore保留，对情绪/爽点/卖点/金手指**反向放行直给**;(3)**新增一整套网文专属轴**(黄金三章留存、爽点密度、章末钩子、毒点规避、品类×平台适配)，这条轴文学清单完全没有，却是"扑街 vs 起飞"的真正分水岭。让人类定爽点/节奏/钩子骨架、AI填空，是当前规避网文AI味的最稳路径。
